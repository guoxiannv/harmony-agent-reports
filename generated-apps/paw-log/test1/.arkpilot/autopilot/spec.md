# 宠迹 · PawLog — 产品规格书

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即HTML设计稿390px = 设备上1170vp物理像素，但ArkTS使用vp单位时1px≈1vp）
- 布局策略：内容区max-width约束在390-420vp内居中，禁止全宽铺满。**注意：Tab导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与body一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在PC浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header+核心卡片）在顶部600vp内可见

## 产品定位

### 价值主张

为养宠家庭打造一款“记录即安心”的轻量级宠物健康管理工具。通过多宠档案、一键健康记录、趋势图表和提醒，帮助用户把零散的日常照护变成可追踪的健康轨迹，让每只毛孩子都拥有属于自己的成长与健康档案。

### 产品类别与核心场景

- 产品类别：生活健康 / 宠物照护工具型 App
- 主要用户：养猫、养狗、养小型宠物的家庭，尤其是多宠家庭
- 核心场景：
  1. 每天随手记录喂食、排便/排尿、体重、用药情况；
  2. 查看体重/饮食/排泄等趋势图表，判断健康变化；
  3. 疫苗、驱虫、体检等周期性提醒到期时收到通知；
  4. 就医后上传病历、处方、化验单并归档；
  5. 家庭成员共享同一组宠物档案，避免重复记录或遗漏。

## 核心任务（Jobs-to-be-Done）

1. **建立并维护多只宠物的专属档案**  
   用户可以添加多只宠物，录入头像、名字、品种、生日、性别、体重、绝育状态、基础病史，并随时编辑。

2. **一键快速记录日常健康行为**  
   通过少量点击完成饮食、排泄、体重、用药四类记录，每种记录包含时间、类型、数量/剂量、备注等必要字段。

3. **查看健康趋势图表**  
   以折线图/柱状图方式展示体重、每日食量、排泄次数、用药周期等趋势，支持按周/月切换。

4. **疫苗与驱虫提醒管理**  
   为每只宠物设置疫苗、驱虫、体检等提醒，支持到期日、提前通知天数、是否重复；到期/逾期时在首页以醒目状态呈现。

5. **就医病历归档**  
   添加就诊记录（医院、医生、症状、诊断、用药），并支持上传最多 9 张图片（病历、化验单、处方），按时间归档。

6. **家庭成员共享**  
   创建/加入家庭组，邀请成员共同管理同一批宠物档案和记录（本期以本地账号映射+家庭成员列表形式呈现，真实云端同步作为非目标）。

## 导航模型

- **主导航**：底部 Tab Bar，固定在屏幕底部，限制宽度与 body 一致，5 个 Tab：
  1. 今日（首页）
  2. 记录
  3. 添加（中央悬浮快捷按钮，视觉突出，不归属于某一页）
  4. 趋势
  5. 我的
- **底部 Tab  my**：入口包括家庭成员、提醒管理、病历归档、设置（本期设置仅占位）。
- **次级导航**：
  - 首页横向宠物切换器（Chips/头像列表）。
  - 记录页顶部时间范围与类型筛选。
  - 趋势页顶部指标切换与周期切换。
- **弹窗路由**：记录编辑、宠物编辑、提醒编辑、就诊编辑、家庭成员邀请均使用底部 Sheet 弹窗承载，避免页面栈过深。

## 数据源策略

- **本地优先**：所有宠物档案、记录、提醒、病历、家庭成员数据优先持久化到 App 本地存储（`AppStorage` / `Preferences` / 本地 JSON 文件），保证离线可用。
- **真实空状态**：首次安装后不应预置任何示例宠物或记录；首页、记录页、趋势页、提醒页、病历页均需提供真实空状态提示与创建入口。
- **家庭成员共享**：家庭成员关系在本期以本地“家庭组”模型实现（创建者、成员昵称/角色），不依赖后端账号体系，真实云端同步列为非目标。
- **图片存储**：病历图片以本地文件路径/URI 形式保存，不上传云端；大图预览使用本地图片源。
- **提醒触发**：依赖系统本地通知能力（`notificationManager`），提醒到期时推送本地通知；不支持远程推送。

## 约束与非目标

### 约束
- 必须遵循 HarmonyOS ArkTS 严格模式。
- UI 必须适配 1272×2756 目标设备，内容区 max-width 390-420vp 居中，底部 Tab 禁止侧边放置。
- 所有语义化交互目标必须提供稳定 ID（`data-ui-id`）以便 QA 与自动化测试定位。
- 颜色以“薄荷绿 + 肉垫粉”为主视觉方向，营造圆润治愈风格。

### 非目标
- 不实现云端账号登录与实时云同步。
- 不实现社交分享/社区/商城功能。
- 不实现 AI 识别食物/便便/化验单功能。
- 不预置演示数据；首次启动为空状态。
- 不实现 Wearable / 平板 / 折叠屏专属适配。

## 验收标准

1. 用户可以成功添加、编辑、删除多只宠物，并查看宠物详情。
2. 用户可以在 3 次点击内完成饮食/排泄/体重/用药任意一种记录。
3. 首页能正确展示当天待处理提醒与最近记录入口。
4. 趋势页能按周/月展示至少体重与饮食两种图表。
5. 提醒列表能区分“已到期/即将到期/正常”状态，并支持新增/编辑/删除。
6. 病历页支持创建就诊记录并上传/查看图片。
7. 家庭成员页支持创建家庭组、查看成员列表、生成邀请信息。
8. 所有关键页面在 1272×2756 设备上首屏关键内容在 600vp 内可见，底部 Tab 始终位于底部且宽度受控。
9. 应用完成构建无错误，静态检查通过。
