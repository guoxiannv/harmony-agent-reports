# 宠迹 · PawLog 产品规格说明书

## Idea Classification

**CONCRETE** — 用户需求清晰，包含核心功能集合、目标用户与视觉方向，可直接进入设计与实现。

## Value Proposition

宠迹 · PawLog 是一款面向多宠家庭的轻量级宠物健康与日常记录工具。它把“今天吃了多少、有没有排便、体重变化、何时驱虫/疫苗”这些碎片化信息，集中在一个圆润治愈的界面里，让铲屎官用最少操作留下最完整的宠生档案，并通过趋势图表和提醒帮助提前发现健康异常。

## Product Category & Primary User Scenario

- **产品类型**：生活健康 / 宠物管理类应用
- **目标用户**：养有多只猫狗、关注宠物健康、希望家庭成员共同记录的年轻宠主
- **主要场景**：
  1. 早晨 30 秒记录每只宠物的饮食与排泄。
  2. 称重后录入体重，观察近 30 天曲线。
  3. 收到疫苗/驱虫到期提醒后预约医院，并把病历归档。
  4. 邀请伴侣/父母加入家庭空间，共享同一套宠生数据。

## Core Jobs-to-be-Done

1. **管理多只宠物档案**：为每只宠物建立头像、昵称、品种、生日、性别、当前体重等基础档案，并在首页快速切换当前关注的宠物。
2. **一键记录日常事件**：通过底部 Tab 或首页大按钮，快速记录饮食、排泄、体重、用药四类事件，支持时间、数量/剂量、备注。
3. **查看健康趋势图表**：按宠物与指标维度展示体重、饮食、排泄等趋势，支持 7/30/90 天周期切换。
4. **疫苗驱虫提醒与就医归档**：为每只宠物设置下一次疫苗/驱虫日期，到期前提醒；记录就医时间、医院、诊断、处方与附件。
5. **家庭成员共享数据**：通过邀请码或本地配对，让多位家庭成员加入同一“家庭”，共同查看与记录。

## Navigation Model

- **主导航**：底部固定 Tab Bar，共 5 个 Tab：首页、爱宠、记录、健康、家人。首页中央悬浮“+”快捷记录按钮作为全局记录入口。
- **二级入口**：宠物详情、新增/编辑表单、趋势页通过按钮或列表项跳转。
- **返回模型**：非 Tab 页面顶部左侧返回按钮，表单保存后返回上一级并刷新列表。
- **首屏策略**：用户首次打开为真实空状态，引导添加第一只宠物；无预设种子数据。

## Data Source Strategy

- **本地持久化**：使用 ArkTS 提供的 `AppStorage` + `PersistentStorage` / `Preferences` 存储宠物档案、记录事件、提醒与病历。不依赖后端服务。
- **首空策略**：默认不提供示例数据，首次使用展示空状态与添加引导；UI 设计稿中的示例数据仅作视觉参考，不作为产品默认数据。
- **家庭成员共享（MVP）**：通过本地生成的邀请码或二维码实现家庭成员加入，同一设备/局域网内共享同一数据源；云端同步不在 MVP 范围。
- **图表数据**：根据本地记录事件实时聚合生成，无离线/在线混合策略。

## Component Structure

- **Pages**：Home、Pets、PetDetail、Records、Trends、Health、Family、AddRecord、AddPet、AddReminder、AddMedical
- **可复用 ArkUI 组件（概念）**：
  - `PetAvatar`：圆形头像，支持图片/默认 paw 占位
  - `RecordTypeChip`：饮食/排泄/体重/用药 四类快捷类型选择
  - `MetricCard`：首页统计卡片（今日喂食次数、排便次数、最新体重、用药次数）
  - `TimelineItem`：记录时间线项
  - `ReminderCard`：疫苗/驱虫提醒卡片，含到期状态
  - `MedicalCard`：就医病历折叠/展开卡片
  - `TrendChart`：基于本地数据渲染的折线/柱状趋势图
  - `FamilyMemberItem`：成员头像 + 权限标识
  - `BottomTabBar`：底部导航，含中央悬浮记录按钮
  - `EmptyState`：空状态插画 + 主文案 + 行动按钮

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即HTML设计稿390px = 设备上1170vp物理像素，但ArkTS使用vp单位时1px≈1vp）
- 布局策略：内容区max-width约束在390-420vp内居中，禁止全宽铺满。**注意：Tab导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与body一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在PC浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header+核心卡片）在顶部600vp内可见

## Acceptance Criteria

1. 首次启动显示空状态，引导用户添加第一只宠物。
2. 用户可以创建、编辑、删除多只宠物档案；删除时二次确认。
3. 用户可为任一宠物一键记录饮食、排泄、体重、用药，并查看时间线。
4. 健康趋势页可按宠物、指标、周期展示图表与摘要。
5. 用户可为宠物设置疫苗/驱虫提醒，到期前后有状态标识；可记录就医病历并查看历史。
6. 家庭页支持邀请成员并展示成员列表（MVP 可用本地模拟）。
7. 所有页面在 390-420vp 宽度容器内居中，底部 Tab 在 5 个主页面保持一致。
8. 构建通过，应用能在模拟器/真机正常启动且无崩溃。

## Non-Goals

- 不在 MVP 内：实时云端同步、宠物社交、AI 健康诊断、电商/商城、智能硬件接入、多语言国际化。
- 不强制接入系统日历/通知，提醒采用应用内状态展示。
- 不提供默认示例数据。
