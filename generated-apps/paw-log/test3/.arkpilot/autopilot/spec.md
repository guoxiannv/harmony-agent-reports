# PawLog 宠迹 · 产品需求规格说明书

## 1. 产品概述

### 1.1 产品名称
- **中文名**：宠迹
- **英文名**：PawLog

### 1.2 价值主张
为养犬、猫、异宠的家庭提供一站式的宠物健康与生活记录工具。通过多宠档案、一键行为记录、健康趋势图表、疫苗驱虫提醒、就医病历归档以及家庭成员共享，让宠物照护从“凭感觉”变为“有依据”，帮助宠物主人在繁忙生活中不遗漏任何一次喂食、驱虫或疫苗，降低健康风险。

### 1.3 产品类别与核心场景
- **产品类别**：宠物健康与生活管理类应用（Pet Health & Care Tracker）。
- **核心场景**：
  1. 早晨起床后，快速记录昨日夜间宠物的排泄与饮食情况。
  2. 兽医问诊前，调出该宠物最近 30 天的体重曲线与用药记录。
  3. 家庭成员中任意一人发现宠物异常后，立即拍照并归档到病历，同时通知其他成员。
  4. 疫苗或驱虫到期前 3 天，主人收到提醒并完成预约。

## 2. 目标用户

- **主要用户**：25-40 岁的城市宠物主人，饲养 1-3 只宠物。
- **次要用户**：与主要用户同住的家庭成员（伴侣、父母），需要查看/协同记录但不一定管理设置。

## 3. 核心 Jobs-to-be-Done

1. **管理多只宠物档案**：为每只宠物建立独立档案，包括头像、昵称、品种、性别、体重、生日、绝育状态、过敏源、性格标签等，方便多宠家庭切换查看。
2. **一键记录日常行为**：通过最少点击完成饮食、排泄、体重、用药四类事件的记录，支持快捷模板（例如“晚餐 50g 猫粮”）。
3. **追踪健康趋势**：以图表形式展示体重、饮水、排泄频次等关键指标的变化趋势，支持按 7 天 / 30 天 / 90 天维度查看。
4. **疫苗与驱虫提醒**：为每只宠物维护疫苗与驱虫时间表，支持按下一次日期生成本地提醒。
5. **归档就医病历**：记录就诊时间、医院、医生、诊断、处方、用药建议，并支持上传多张图片（检查单、化验单、患处照片）。
6. **家庭成员共享**：通过邀请码或关联账号让家庭成员加入同一宠物空间，数据实时同步，区分记录人身份。

## 4. 页面结构

### 4.1 Tab 页面（底部导航）

| 页面 ID | 页面名称 | 主要职责 |
|---------|----------|----------|
| `home-page` | 首页 | 展示今日待办提醒、最近记录入口、宠物快捷切换、核心一键记录按钮。 |
| `records-page` | 记录 | 按时间轴展示所有宠物的饮食/排泄/体重/用药记录，支持筛选与搜索。 |
| `health-page` | 健康 | 展示健康趋势图表、疫苗/驱虫计划、病历列表入口。 |
| `family-page` | 家庭 | 家庭成员列表、邀请成员、宠物共享权限管理。 |
| `profile-page` | 我的 | 当前用户信息、设置、多宠档案管理入口、数据导出/备份入口。 |

| 页面 ID | 页面名称 | 主要职责 |

| `chart-detail-page` | 图表详情 | 某只宠物某类指标的放大图表与数据列表。 |
| `reminder-editor-page` | 提醒编辑 | 新增/编辑疫苗或驱虫提醒。 |
| `medical-record-editor-page` | 病历编辑 | 新增/编辑就医病历，包括图片附件。 |

### 4.3 浮层 / 弹窗

- 宠物选择器（`pet-picker-sheet`）：在记录前选择目标宠物。
- 记录类型选择器（`record-type-sheet`）：选择饮食 / 排泄 / 体重 / 用药。
- 日期/时间选择器（`date-time-picker`）。
- 确认删除弹窗（`confirm-delete-dialog`）。

## 5. 导航模型

- **主导航**：底部固定 TabBar，包含 5 个 Tab：首页、记录、健康、家庭、我的。
- **次级导航**：Tab 页面内通过卡片、列表项、按钮进入非 Tab 全页。
- **返回模型**：非 Tab 页面通过左上角返回按钮或系统手势返回上一个 Tab 页面。
- **模态操作**：选择器与确认弹窗以底部 Sheet 或居中 Dialog 形式呈现。

## 6. 数据策略

- **数据源**：应用优先使用本地持久化存储（App Storage / 本地数据库）存储宠物档案、记录、提醒、病历、家庭成员关系。家庭成员共享通过云端账号空间同步实现。
- **初始状态**：首次启动为真实空状态，不强制预置示例数据。用户首次进入首页时引导添加第一只宠物。
- **离线能力**：核心记录与查看功能在无网络时可用；家庭成员同步在有网络连接时自动同步。
- **图片存储**：病历图片保存到应用私有目录，支持从相册选择或拍照。
- **提醒**：提醒时间存储在本地，由系统本地通知触发；云端仅备份提醒元数据。

## 8. 功能需求细则

### 8.1 多宠档案
- 支持新增、编辑、删除宠物档案。
- 必填：昵称、物种（猫/狗/其他）、性别、生日/到家日。
- 选填：头像、品种、毛色、体重、绝育状态、芯片号、过敏源、性格标签、备注。

### 8.2 一键记录
- 首页提供一键记录入口，点击后先选择宠物（如未指定），再选择记录类型。
- 饮食记录：食物名称、食量、单位、时间、备注。
- 排泄记录：排泄类型（大便/小便/呕吐/其他）、外观状态、时间、备注。
- 体重记录：体重数值、单位、时间、备注。
- 用药记录：药品名称、剂量、时间、是否已服用、备注。
- 支持从记录页和首页快捷创建最近一次的同类型记录。

### 8.3 健康趋势图表
- 在健康页展示最近 7 天默认视图。
- 支持切换 7 天 / 30 天 / 90 天。
- 支持查看单只宠物的体重、饮水、排泄频次趋势。
- 点击迷你图表进入图表详情页，查看完整数据列表。

### 8.4 疫苗与驱虫提醒
- 按宠物维护疫苗与驱虫计划。
- 每种提醒包含：类型名称、计划日期、重复周期、已/未完成状态。
- 到期前 3 天与当天分别提醒一次。
- 首页展示今日/即将到期的提醒。

### 8.5 就医病历归档
- 病历包含：就诊日期、医院/诊所、医生、主诉、诊断、治疗方案、用药、费用、备注、图片附件。
- 支持查看详情、编辑、删除。

### 8.6 家庭成员共享
- 每个家庭创建一个共享空间，空间内包含多只宠物与多名成员。
- 成员角色：管理员（可管理宠物、成员、设置）与普通成员（可记录/查看）。
- 通过邀请码或链接邀请成员。
- 记录中展示记录人身份。

## 9. 非目标

- 不提供电商、宠物社交、宠物领养、宠物定位硬件联动。
- 不提供与医院系统的在线预约或处方直开功能。
- 不提供 AI 诊断建议，仅做数据记录与趋势展示。
- 不内置默认示例宠物数据，首次启动为空状态。

## 10. 验收标准

1. 用户可成功创建并管理多只宠物档案，档案间切换不丢失状态。
2. 用户可在 3 步内完成一次饮食/排泄/体重/用药记录。
3. 健康图表能正确按时间维度聚合并渲染趋势曲线。
4. 疫苗/驱虫提醒在到期前 3 天与当天触发本地提醒。
5. 病历可保存文字与多张图片，详情页可完整展示。
6. 家庭成员加入同一空间后，能互相看到记录与更新。
7. 应用在 1272×2756 vp 手机上正常渲染，关键首屏内容在顶部 600vp 内可见。

## 11. 待澄清问题

- 是否需要接入推送通知服务，还是仅使用本地提醒？
- 家庭成员共享是否使用第三方账号体系，还是以本地匿名空间 + 邀请码方式实现？
- 是否需要在图表中展示目标体重区间或参考线？

## 12. Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即HTML设计稿390px = 设备上1170vp物理像素，但ArkTS使用vp单位时1px≈1vp）
- 布局策略：内容区max-width约束在390-420vp内居中，禁止全宽铺满。**注意：Tab导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与body一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在PC浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header+核心卡片）在顶部600vp内可见

## 13. 视觉方向（由 Design 阶段细化）

- 主色调：薄荷绿 + 肉垫粉，营造清新、治愈、温润的氛围。
- 风格关键词：圆润、柔和、卡片化、治愈系、宠物友好。
- 图标与插图以圆润 paw/肉垫、卡通宠物头像为主，避免尖锐几何。
