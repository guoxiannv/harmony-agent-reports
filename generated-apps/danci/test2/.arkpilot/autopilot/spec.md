# 产品规格文档：童词卡（Kids Word Cards）

## 1. 产品定位与价值主张

**产品名称**：童词卡（Kids Word Cards）  
**产品类型**：儿童定制化单词复习工具（HarmonyOS 原生应用）  
**核心价值**：让孩子以游戏化的“抽卡—翻转—自评”方式，高效复习当前阶段学过的单词，通过即时反馈建立正向记忆循环。

## 2. 主要用户场景

- **场景 A：课后复习**：孩子在完成当日单词学习后，打开 App 进入复习模式，依次浏览、自评，巩固记忆。
- **场景 B：碎片时间复习**：利用课间或睡前短暂时间，快速过一遍当前词库。
- **场景 C：阶段过关**：当所有卡片都被标记为“认识”后，显示完成状态，给予成就感反馈。

## 3. 核心任务（Jobs-to-be-Done）

1. 从单词库加载孩子“当前阶段学过的单词”。
2. 以卡片堆形式展示当前单词，中央单张大卡片，正面为英文单词，背面为中文释义与音标。
3. 点击卡片触发 3D 翻转动画，切换正反面。
4. 提供“认识”与“不认识”两个自评按钮。
5. “认识”：将单词移出卡片堆，记录正确数，并展示下一张卡片。
6. “不认识”：将卡片放回队列底部，稍后再次复习。
7. 实时显示当前复习进度（剩余卡片数 / 已认识数）。
8. 当卡片堆清空时，显示完成/过关状态。

## 4. 页面结构

- **StudyPage（复习页）**：唯一主页面，承载卡片堆、翻转卡片、自评按钮、进度统计。

## 5. 导航模型

- 单页面应用，无底部 Tab 导航。
- 启动后直接进入 StudyPage。
- 完成后在同一页展示完成覆盖层/状态，提供“再复习一次”入口。

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 6. 数据源策略

- 初始数据源：应用内置的“单词库”，按学习阶段过滤后加载。
- 本次不实现用户自定义词库、登录同步或后端服务。
- 首次运行直接显示当前阶段单词列表；不强制要求预置示例数据作为功能种子，但允许 HTML 设计稿中使用视觉示例数据。
- 状态持久化：本次仅要求运行时状态（当前队列、已认识数），不要求跨会话持久化。

## 7. 组件结构

- `StudyPage`：页面容器，管理词队列、当前卡片索引、翻转状态、统计。
- `FlashCard`：3D 翻转卡片组件，支持正面（英文）与背面（释义+音标）。
- `ProgressHeader`：顶部进度条与统计（剩余/已认识）。
- `ActionBar`：底部“认识 / 不认识”按钮组。
- `CompletionOverlay`：完成后的覆盖反馈组件。

## 8. 验收标准

- [ ] 启动后自动加载当前阶段单词队列。
- [ ] 中央显示单张卡片，正面仅展示英文单词。
- [ ] 点击卡片触发 3D 翻转动画，展示中文释义与音标。
- [ ] 卡片下方固定“认识”“不认识”两个按钮。
- [ ] 点击“认识”：当前卡片从堆顶移除，已认识数 +1，显示下一张。
- [ ] 点击“不认识”：当前卡片移动到队列末尾，保持当前展示下一张。
- [ ] 进度头部实时更新剩余卡片数与已认识数。
- [ ] 队列清空后显示完成状态，并提供重新开始按钮。
- [ ] 目标设备上首屏 Header + 核心卡片在顶部 600vp 内可见。

## 9. 非目标（Non-Goals）

- 不支持用户登录、云端同步、学习记录历史。
- 不支持发音播放、拼写输入、例句展示。
- 不支持多阶段选择、词库管理、家长后台。
- 不要求跨会话持久化学习进度。

## 10. 待确认问题

1. 单词库是静态 JSON 文件还是应用内资源？（建议：静态 JSON 资源）
2. 是否需要阶段选择入口，还是默认唯一阶段？（建议：默认唯一阶段，后续可扩展）
3. 完成状态是否展示总词数、正确率等统计？（建议：展示总词数与已认识数）
