# Spec: 孩子定制化记单词 App

## 1. Value Proposition
为学龄儿童提供一款专注、无干扰的单词复习工具。App 从本地单词库中加载孩子当前学习阶段的单词，通过 3D 翻转卡片游戏化的方式帮助他们复习，自动记录认识/不认识的结果，巩固记忆并让孩子（及家长）清晰看到每日复习进度。

## 2. Product Category & Primary User Scenario
- **产品类别**：教育类 / 儿童学习工具（单机/本地优先）。
- **主要用户场景**：
  1. 孩子在课后打开 App，复习当天/当周已学过的单词。
  2. 孩子通过“认识 / 不认识”反馈，让系统决定哪些单词需要再次复习。
  3. 家长/孩子查看本次复习结果（已掌握数量、需复习数量），形成正向反馈。

## 3. Core Jobs-to-be-Done
1. **加载当前阶段单词**：从本地单词库读取孩子已学过的单词列表，作为本次复习卡片堆。
2. **展示复习卡片**：界面中央一张卡片，正面显示英文单词；背面显示中文释义与音标。
3. **翻转卡片学习**：点击卡片触发 3D 翻转动画，在单词与释义之间切换。
4. **反馈记忆结果**：卡片下方提供“认识”和“不认识”两个按钮。
   - 点击“认识”：将当前单词移出卡片堆，记录正确数。
   - 点击“不认识”：将当前单词放回队列底部，稍后再次复习。
5. **查看进度**：实时显示剩余卡片数、已认识数量，本轮复习完成后展示总结。

## 4. Page Structure
- **首页 / 复习页 (`review-page`)**：唯一核心页面。
  - Header：App 名称、本轮进度（剩余/总数）。
  - 中央卡片区：3D 翻转卡片（正面英文 / 背面释义+音标）。
  - 操作区：两个主按钮（认识 / 不认识）。
  - 进度摘要：已认识数量、待复习数量。
- **完成状态（同页内嵌）**：当卡片堆为空时，在卡片区展示“本轮复习完成”总结，并提供“再来一轮”入口。

## 5. Navigation Model
- 单页面应用，无底部 Tab、无侧栏导航。
- 所有交互发生在 `review-page` 内；完成状态通过条件渲染切换内容。

## 6. Data Source Strategy
- 数据存储：优先使用本地持久化能力（HarmonyOS `relationalStore` / `preferences` / 本地 JSON 文件），离线可用。
- 单词库：预置本地单词库文件（JSON / SQLite），包含 `word`、`phonetic`、`meaning`、`stage/level` 字段。
- 首运行状态：单词库作为应用资源内置，但孩子的“已学阶段”与“本轮复习状态”首运行为空；App 默认进入当前阶段为空的首屏，提供选择/加载当前阶段入口。本阶段不强制要求真实同步后端。
- 学习记录：仅记录本轮内的认识数、不认识数；不强制记录跨轮历史。

## 7. Component Structure
- `ReviewPage`：页面容器，管理本轮状态（队列、索引、计数）。
- `Flashcard`：可翻转卡片组件，接收 `word` 对象，支持点击翻转。
- `ProgressHeader`：标题与进度指示。
- `ActionBar`：认识/不认识按钮组。
- `CompletionPanel`：本轮完成后的总结面板。

## 8. Target Device
- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 9. Constraints & Non-Goals
- **约束**：
  - 界面需针对儿童操作优化：按钮大、反馈明确、动画柔和。
  - 3D 翻转动画必须在 300-500ms 之间，使用 ease-in-out 缓动。
  - 点击“不认识”后必须将单词放到队列末尾，不能随机打乱。
- **非目标**：
  - 不实现用户登录、多账户、云端同步。
  - 不实现复杂的单词拼写/听写功能。
  - 不实现社交分享或排行榜。
  - 不需要每轮都重置真实学习进度历史。

## 10. Acceptance Criteria
- [ ] App 启动后从本地单词库加载当前阶段单词列表。
- [ ] 首屏默认展示队列第一张卡片的英文单词正面。
- [ ] 点击卡片执行 3D 翻转动画，显示中文释义和音标。
- [ ] 点击“认识”：当前单词从队列移除，已认识计数 +1，展示下一张卡片。
- [ ] 点击“不认识”：当前单词移至队列末尾，计数不变，展示下一张卡片。
- [ ] 当队列清空时，展示本轮复习完成总结（已认识 / 待复习数量）。
- [ ] 提供“再来一轮”按钮，使用相同单词库重新生成队列。
- [ ] 在 1272×2756 手机上首屏 Header + 卡片位于 600vp 以内。

## 11. Open Questions
1. 是否需要家长模式/设置页来选择学习阶段？（当前版本使用默认当前阶段。）
2. 是否需要发音按钮（TTS）辅助孩子学习？（当前版本不包含，可作为后续增强。）
3. 是否需要动画音效或触觉反馈？（当前版本不包含。）
