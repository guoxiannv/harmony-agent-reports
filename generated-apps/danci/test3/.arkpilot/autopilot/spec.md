# Spec: 儿童定制化单词复习卡（WordFlip Kids）

## 1. Idea Classification

**CONCRETE.** 用户已给出明确的产品形态、核心交互与数据行为：从单词库加载学过的单词，以卡片形式复习；卡片正面英文、背面中文释义+音标；点击卡片 3D 翻转；下方「认识 / 不认识」按钮分别将单词移出卡片堆或放回队列底部，并记录正确数。

## 2. Value Proposition

为学龄儿童（6-12 岁）提供一个专注、低认知负荷的单词复习工具。通过类实体卡片的 3D 翻转记忆法，让孩子在没有家长监督的情况下也能独立完成当前阶段学过的单词复习，并通过「认识 / 不认识」的自我反馈形成可追踪的学习闭环。

## 3. Product Category & Primary User Scenario

- **产品类别**：教育类 / 儿童学习辅助工具
- **主要用户场景**：孩子每天利用碎片时间（如放学后、睡前）复习当天/当周在学校或 App 内学过的单词。App 打开后直接进入复习卡片堆，无需复杂导航。

## 4. Core Jobs-to-be-Done

1. **加载当前阶段单词**：从本地单词库读取孩子已学过的单词列表，作为今日复习卡片堆。
2. **展示并翻转单词卡片**：中央卡片正面显示英文单词，点击后以 3D 动画翻转到背面，显示中文释义和音标。
3. **自我评估与队列调度**：孩子根据记忆情况点击「认识」或「不认识」；「认识」移除卡片并累加正确数，「不认识」将卡片移到队列底部稍后复现。
4. **查看复习进度**：顶部显示当前剩余卡片数与已答对数，让孩子对进度有掌控感。
5. **完成复习**：卡片堆清空后展示完成状态与本次正确数，并提供重新开始的入口。

## 5. Page Structure

| 页面 | 类型 | 职责 |
|------|------|------|
| `review` | 主页面 / 唯一核心页 | 单词卡片复习主流程 |
| `completion` | 结果页（同路由内 Sheet/覆盖层） | 复习完成后展示统计与重学入口 |

**说明**：本版本为单一核心流程的最小可用产品。不引入单词本管理、登录、设置等多页结构。

## 6. Navigation Model

- **启动后直接进入** `review` 页面，无底部 Tab、无侧边抽屉。
- `completion` 作为 `review` 页面上的结果覆盖层或 Sheet，不单独配置路由。
- 顶部仅显示静态页面标题与进度统计，无返回按钮。

## 7. Data Source Strategy

- **数据源**：本地持久化单词库（`AppStorage` / 轻量级首选项 / 本地 JSON 资源，由实现阶段决定）。
- **首次运行状态**：允许单词库为空。若当前阶段没有可复习单词，页面展示空状态（empty state），提示「还没有可复习的单词哦」。
- **不默认填充演示数据**：除非用户后续明确要求，否则设计阶段的 HTML 中任何示例单词仅用于视觉展示，不构成产品功能需求。
- **复习算法**：简单队列（FIFO）。点击「认识」出队；点击「不认识」移至队尾。
- **统计字段**：`totalCount`（本轮总数）、`remainingCount`（剩余卡片数）、`correctCount`（已认识数）。

## 8. Component Structure

- `ReviewPage`：页面容器，管理复习队列与统计状态。
- `Flashcard`：3D 翻转卡片组件，正面 / 背面状态由父组件控制。
- `ProgressHeader`：顶部标题、剩余卡片数、已答对数。
- `ActionBar`：「认识」与「不认识」两个主要操作按钮。
- `CompletionOverlay`：完成后的结果覆盖层。
- `EmptyState`：无可复习单词时的空状态。

## 9. Constraints & Non-Goals

- **必须实现**：3D 翻转动画、认识/不认识按钮行为、队列调度、正确数统计。
- **不做**：单词编辑/新增、发音播放、学习日历、家长后台、云端同步、等级/积分系统。
- **目标平台**：HarmonyOS 普通手机，ArkTS + ArkUI。

## 10. Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 11. Acceptance Criteria

- [ ] 启动后进入复习页面，自动加载当前阶段已学单词队列。
- [ ] 页面首屏可见标题、进度（剩余 / 答对）和中央卡片。
- [ ] 点击卡片正面触发 3D 翻转动画，翻到背面显示中文释义和音标。
- [ ] 再次点击卡片可从背面翻回正面。
- [ ] 点击「认识」：当前卡片移出队列，correctCount +1，显示下一张卡片；若队列为空则进入完成状态。
- [ ] 点击「不认识」：当前卡片移至队列底部，remainingCount 不变，显示下一张卡片。
- [ ] 当当前阶段无单词时，显示空状态文案。
- [ ] 完成状态显示本轮答对数与「再学一次」入口，点击后重新加载当前阶段队列。
