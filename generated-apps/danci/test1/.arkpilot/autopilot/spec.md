# 产品规格书：小小单词卡

## 产品定位

**产品名称**：小小单词卡  
**产品类别**：儿童教育类（K12 英语单词复习工具）  
**目标用户**：学龄儿童（6-12 岁）及其家长  
**核心场景**：孩子在每日学习后，打开 App 复习当天或当前阶段已学过的单词。通过翻转卡片的游戏化交互，强化「英译中」记忆，同时让家长/孩子自己掌握复习进度。

## 价值主张

把背单词变成一场像翻牌游戏一样的小挑战：
- 只看英文，孩子主动回忆中文释义，再翻卡验证；
- 「认识 / 不认识」两个按钮让复习有即时反馈；
- 不认识的词会自动回到队尾再次复习，确保真正记住；
- 完成一轮后给出正向鼓励，让孩子愿意继续。

## 核心 Jobs-to-be-Done

1. **加载当前阶段的单词**：打开 App 即可看到孩子正在学习的单词集合，无需手动输入。
2. **主动回忆并验证**：通过「正面英文 → 点击翻卡 → 背面中文+音标」的流程，完成一次完整的记忆提取。
3. **标记掌握程度**：孩子自己判断「认识」或「不认识」，系统据此调整复习队列。
4. **循环复习不熟的词**：不认识的单词不会消失，而是回到队列底部，让孩子在学习会话中再次遇到。
5. **查看学习成果**：一轮复习结束后，显示本轮正确数 / 总词数，给予完成感。

## 页面结构

| 页面 | 页面 ID | 路由 | 说明 |
|------|---------|------|------|
| 复习页 | `review-page` | `pages/Review` | 核心单词卡片复习页面，包含翻卡与掌握标记。 |

## 导航模型

本应用为单页面工具型应用，无需底部 Tab 或抽屉导航。
- 启动即进入复习页。
- 一轮完成后在同一页面显示结果浮层/占位，点击「再学一次」重新加载当前阶段单词队列。

## 数据结构策略

- **当前阶段单词库**：由外部学习系统/家长配置的 JSON 数组提供。首屏为「真实空状态」；若当前阶段没有单词，则显示空状态提示「还没有要复习的单词哦」。
- **运行时队列**：App 启动时从数据源拷贝一份 `reviewQueue`，复习过程中原地操作（移出/移到队尾）。
- **学习记录**：仅记录本轮 `correctCount` 与 `totalCount`，不持久化历史记录（超出当前范围）。
- **空状态**：真实空状态，不允许把 HTML 示例数据默认当作种子数据。

## 组件结构

- **复习页 (`ReviewPage`)**
  - 顶部栏：显示页面标题、本轮进度（如 3/10）。
  - 进度指示器：环形或条形，展示当前卡片在队列中的位置。
  - 3D 翻转卡片 (`FlashCard`)：
    - 正面：英文单词（大号展示）。
    - 背面：中文释义 + 音标。
  - 操作按钮组：
    - 「不认识」按钮（将当前卡片移到队尾）。
    - 「认识」按钮（将当前卡片移出队列并正确数 +1）。
  - 完成状态 (`CompletionState`)：卡片队列为空时显示本轮统计与「再学一次」按钮。
  - 空状态 (`EmptyState`)：数据源为空时提示无复习内容。

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px ≈ 1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。**
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 验收标准

1. 启动应用后进入复习页，自动加载当前阶段单词队列。
2. 当前卡片正面仅展示英文单词；点击卡片后播放 3D 翻转动画，展示中文释义和音标。
3. 再次点击已翻转的卡片可翻回正面（或保持背面直到操作按钮被点击，由实现决定，但至少支持一次完整翻转）。
4. 点击「认识」：当前单词从队列移除，`correctCount` 增加 1，自动展示下一张卡片。
5. 点击「不认识」：当前单词移到队列底部，自动展示下一张卡片（原队首单词）。
6. 当队列为空时，显示完成状态，展示「本轮正确数 / 总词数」与「再学一次」按钮；点击按钮重新从数据源加载队列。
7. 数据源为空时显示空状态文案与占位图/图标。
8. 所有按钮与卡片必须满足 44vp × 44vp 的最小触控区域。
9. 所有交互元素具有稳定的 `data-ui-id`（ArkTS 中对应 `.id(...)`），用于后续测试断言。

## 非目标

- 不在本阶段实现用户登录、账号体系或多设备同步。
- 不在本阶段实现发音/语音播放功能。
- 不在本阶段实现历史学习记录、错题本或艾宾浩斯记忆算法。
- 不在本阶段实现自定义单词录入或编辑功能。

## 待确认问题

1. 单词数据源格式是否固定为本地 JSON 文件？是否需要从网络 API 获取？
2. 当前阶段单词是否按课程/单元分组，还是统一一个列表？
3. 是否需要在卡片上展示例句或图片（超出中文释义+音标之外）？
4. 一轮复习结束后，「再学一次」是重新复习全部单词还是仅复习本轮标记为不认识的单词？
