# 平安扣 Pinankou - 产品规格

## Value Proposition

“平安扣”是一款面向企业一线班组、安全员及现场管理人员的移动安全管理工具。它将安全事项记录、风险隐患上报、巡检打卡、应急联系人、整改进度跟踪和统计看板整合在一个 App 中，帮助团队把“安全”从被动响应变成主动防控，让每一次隐患都能被记录、被跟进、被闭环。

## Product Category & Primary User Scenario

- **产品类别**：B2B 轻量级工具 / 企业现场安全管理 SaaS 移动端
- **核心场景**：
  1. 安全员在现场巡检时拍照打卡、记录隐患并指派整改人。
  2. 班组长每日班前/班后快速记录安全事项、查看待整改项。
  3. 发生异常时，一键呼叫应急联系人并查看整改进度。
  4. 管理层通过统计看板了解隐患趋势、整改完成率和巡检覆盖率。

## Core Jobs-To-Be-Done

1. **安全事项记录**：快速创建安全事件（时间、地点、类型、描述），支持文字、照片和语音入口。
2. **风险隐患上报**：发现隐患后立即拍照、选择风险等级、定位、提交并自动通知责任人。
3. **巡检打卡**：按路线/点位完成巡检，扫码或手动打卡，记录异常并生成巡检记录。
4. **应急联系人**：维护紧急联系人列表，支持一键拨号、按角色/分组筛选。
5. **整改进度跟踪**：查看隐患整改任务的状态、截止时间和责任人，支持催办和验收。
6. **统计看板**：展示隐患数量、整改进度、巡检覆盖率和趋势图表，支持按时间/区域筛选。

## Navigation Model

- **底部 Tab 导航**：工作台、隐患、巡检、联系人、统计、我的共 6 个 Tab。日常批量验证场景下保持稳定入口。
- **子页面栈**：隐患详情、隐患编辑通过路由跳转进入，左侧返回按钮。
- **快捷操作**：工作台提供“新建隐患”“开始巡检”“一键呼救”等快捷入口。

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px ≈ 1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## Data Source Strategy

- **本地优先 + 云端同步**：核心数据（隐患、巡检记录、联系人、安全事项）本地持久化，具备网络时增量同步到后端。
- **空首屏**：首次启动不预设示例数据，所有列表均展示真实空状态，引导用户创建第一条记录。
- **文件与媒体**：拍照/选图先保存到本地沙盒，上传成功后替换为远端 URL；离线时保留本地路径。
- **统计看板**：聚合本地数据实时计算，关键指标缓存 5 分钟。
- **权限**：相机、定位、麦克风、通讯录按需申请，首次使用对应功能时弹窗提示。

## Component Structure

- **全局**：App 入口、底部 TabBar、导航栏、加载/空状态/错误提示、Toast。
- **原子组件**：安全状态徽章、风险等级标签、进度环、数据卡片、联系人头像、搜索栏。
- **业务组件**：
  - `HazardCard`：隐患摘要卡片
  - `HazardForm`：隐患表单
  - `PatrolRouteCard`：巡检路线卡片
  - `PatrolCheckpointItem`：巡检点位项
  - `ContactCard`：联系人卡片
  - `StatSummary`：统计指标卡片
  - `TrendChart`：趋势图
- **页面组件**：Home、Hazard、HazardDetail、HazardEdit、Patrol、Contacts、Stats、Profile。

## Constraints

- 必须适配 HarmonyOS ArkTS / ArkUI 开发规范。
- 优先使用 HarmonyOS 原生能力（相机、定位、通讯录）与统一存储。
- 蓝绿色安全感配色贯穿全局，信息层级清晰，适合日常批量验证。
- 不引入第三方统计 SDK；图表使用系统自绘或轻量自定义组件。
- 不处理企业级权限 RBAC，仅区分“普通用户”与“管理员”两种角色开关。

## Non-Goals

- 不实现实时视频监控接入。
- 不实现复杂工作流引擎或审批链。
- 不实现多租户/多企业切换（单企业版）。
- 不要求 AI 自动识别隐患图片内容。

## Acceptance Criteria

1. 用户可在 3 步内完成一条隐患上报（拍照 → 选等级 → 提交）。
2. 巡检打卡支持手动打卡与异常拍照，打卡记录可离线保存。
3. 应急联系人支持一键拨号，联系人分组可见。
4. 统计看板至少展示隐患总数、整改完成率、巡检覆盖率 3 个核心指标。
5. 列表空状态有明确引导文案与主操作按钮。
6. 应用在目标分辨率手机上可构建通过，并在模拟器/真机正常运行。

## Open Questions

1. 是否需要离线地图或仅使用定位坐标？（当前默认仅定位坐标）
2. 是否需要扫码打卡？（当前保留入口但可降级为手动打卡）
3. 整改任务是否需要消息推送？（当前通过本地通知实现催办提醒）
