## 一句话价值主张
平安扣是一款面向一线安全员与班组负责人的移动端安全管理工具，把“发现—上报—整改—复核”闭环装进手机，让每一次巡检和风险处理都可记录、可追踪、可量化。

## 产品类别与核心使用场景
- **产品类别**：B2B / 安全生产工具型 App（可覆盖工地、园区、物业、制造车间等场景）
- **主要用户**：安全员、班组长、现场负责人
- **核心场景**：
  1. 日常巡检：按点位逐项检查，拍照/备注，打卡留痕。
  2. 隐患发现：发现风险后快速上报、定级、派给整改人。
  3. 整改闭环：被指派人员收到任务、提交整改、发起复核。
  4. 应急响应：一键拨打应急联系人，查看职责分工。
  5. 管理看板：统计隐患数、整改率、巡检完成率，用于例会汇报。

---

## 核心 Jobs-to-be-Done

1. **记录安全事项**  
   用户能快速新增一条安全事项（如设备异常、人员违规、消防问题），填写标题、类型、地点、描述、照片，并可关联到具体巡检任务。

2. **上报风险隐患**  
   对隐患进行分级（低/中/高/重大）和分类（消防/电气/机械/环境/人员等），指定整改责任人与截止日期，系统生成隐患编号。

3. **巡检打卡**  
   用户按巡检计划逐项确认，支持正常/异常状态；异常自动进入隐患池。打卡记录包含时间、定位、巡检人与备注。

4. **管理应急联系人**  
   维护应急联系人列表（姓名、职务、电话、职责），支持一键拨号与搜索；紧急情况下可在 3 秒内触达。

5. **跟踪整改进度**  
   隐患按“待整改 → 整改中 → 待复核 → 已关闭”流转；责任人与复核人分别可操作并留痕，支持催办。

6. **查看统计看板**  
   首页提供今日巡检、待处理隐患、逾期风险、整改率等关键指标，以及趋势/分类图表，支持按周/月切换。

## 导航模型
- **主导航**：底部固定 Tab Bar，5 个入口：首页、巡检、隐患、应急、我的。
- **二级导航**：
  - 首页：无子导航，以卡片和快捷入口组织。
  - 巡检页：顶部分段控制器“今日/待办/历史”。
  - 隐患页：顶部分段控制器“全部/待整改/整改中/待复核/已关闭” + 筛选/排序。
  - 应急页：顶部搜索 + 按职责分组/字母排序。
  - 我的：列表式设置入口。
- **模态与表单**：新建/编辑/详情以全屏页面或底部 Sheet 形式呈现；破坏性操作使用二次确认弹窗。

## Target Device
- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 数据源策略
- 默认首运行为真实空状态，不预置种子数据。
- 所有数据均在本地持久化（AppStorage / 本地 JSON / 轻量数据库），不依赖后端即可运行。
- 支持导出/导入或后续接入云端，但不在首版范围内。
- UI 设计稿中的示例数据仅作视觉展示，不作为默认产品数据。

## 约束
- 视觉：蓝绿色安全感配色，信息层级清晰，适合日常批量验证。
- 性能：列表页支持 100+ 条记录流畅滚动；图片懒加载/压缩缩略图。
- 无障碍：关键操作目标最小 44×44vp；图标需有语义 title；状态变化有明确颜色+文字双重提示。
- 离线优先：所有写入先落地本地，网络恢复后可后续同步（同步逻辑不在首版）。

## 非目标
- 不在首版实现：用户权限体系、实时多人协作、云端同步、AI 风险识别、电子签章。
- 不预置演示账号或示例项目数据。

## 验收标准
1. 能完成 5 个 Tab 的页面切换与基本空状态展示。
2. 能新增安全事项并出现在隐患列表。
3. 能创建巡检任务并逐项打卡。
4. 能维护应急联系人并实现一键拨号调用系统拨号。
5. 隐患状态可正常流转并在看板实时刷新统计。
6. 构建通过，运行在目标设备上首屏渲染正常。

## 开放问题
- 是否需要定位真实 GPS 或允许手动输入地点？（建议首版允许手动输入，定位作为增强）
- 隐患分级是否由上报人决定，还是支持复核人调整？（建议首版上报人定级，复核人可调整）
- 是否需要拍照或允许从相册选择？（建议支持拍照 + 相册多选，首版最多 6 张）
