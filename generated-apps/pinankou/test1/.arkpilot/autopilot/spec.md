# 平安扣 Pinankou - 产品需求规格说明书

## 1. 产品概述

### 价值主张
"平安扣" 是一款面向企业/园区/物业安全管理人员与一线巡检员工的移动端安全管理工具。它以"蓝绿色安全感"为视觉基调，帮助团队把安全事项、风险隐患、巡检打卡、应急联系人和整改进度集中在一个轻量、可信的入口中，降低安全事故响应延迟，提高日常安全运营的透明度与闭环率。

### 产品类别与主要使用场景
- **产品类别**: B2B / 企业安全生产管理工具（移动端 App）。
- **主要用户场景**:
  1. 一线员工在巡检过程中发现隐患，即时拍照、定位、上报。
  2. 安全负责人每日查看待整改事项、统计看板，分派责任人并跟踪进度。
  3. 紧急情况下，员工快速拨打应急联系人电话或发送求助信息。
  4. 管理人员通过统计看板评估安全健康度与整改闭环率。

### 目标用户
- **一线巡检员 / 操作工**: 需要快速打卡、上报隐患、查看安全事项。
- **安全主管 / 班组长**: 需要分派整改、跟踪进度、查看统计。
- **应急联系人**: 通过电话/短信被触达的救援或管理人员。

## 2. 核心工作（Core Jobs-to-be-Done）

1. **记录安全事项**: 用户可以创建、编辑、查看安全事项（如消防通道检查、设备保养提醒），支持标题、分类、优先级、负责人、截止日期和状态。
2. **上报风险隐患**: 用户可以快速上报隐患，包含位置、风险等级、描述、照片/附件，并进入整改流程。
3. **巡检打卡**: 用户按检查点逐项确认或拍照留痕，完成一次巡检任务并生成记录。
4. **管理应急联系人**: 用户可以维护应急联系人列表（姓名、关系/角色、电话），支持一键拨打或发送短信。
5. **跟踪整改进度**: 用户可以看到隐患/事项的整改进度（待处理、处理中、待验收、已完成），并更新状态与备注。
6. **查看统计看板**: 用户通过图表/数字卡片查看隐患数量、整改完成率、巡检完成率、逾期事项等关键指标。

## 3. 页面结构（Page Structure）

| 页面 ID | 页面名称 | 说明 |
|---------|----------|------|
| `home-page` | 首页 / 工作台 | 汇总今日待办、关键指标入口、快捷操作（隐患上报、巡检打卡） |
| `safety-items-page` | 安全事项 | 安全事项列表、筛选、新增/编辑事项 |
| `hazards-page` | 风险隐患 | 隐患列表、筛选、上报入口、整改进度 |
| `inspections-page` | 巡检打卡 | 巡检任务列表、检查点打卡、历史记录 |
| `emergency-page` | 应急联系人 | 联系人列表、一键拨号/短信、新增/编辑 |
| `stats-page` | 统计看板 | 图表、指标卡片、趋势 |
| `item-form-sheet` | 事项表单 | 新增/编辑安全事项的底部表单或独立页 |
| `hazard-form-sheet` | 隐患表单 | 上报/编辑隐患的底部表单或独立页 |
| `contact-form-sheet` | 联系人表单 | 新增/编辑应急联系人的底部表单或独立页 |

## 4. 导航模型

- **主导航**: 底部 Tab 栏，5 个一级入口：
  - 首页（`tab-home`）
  - 隐患（`tab-hazards`）
  - 巡检（`tab-inspections`）
  - 统计（`tab-stats`）
  - 我的（`tab-profile`）
- **次级导航**:
  - 首页快捷入口进入"安全事项"、"隐患上报"、"巡检打卡"、"应急联系人"。
  - 隐患/事项/联系人列表通过 Floating Action Button（FAB）或顶部"新增"按钮进入表单。
  - 统计页支持顶部时间维度切换（本周/本月/本季度）。
- **返回/关闭**: 表单页通过左上角返回或底部"取消"关闭；模态表单支持下滑关闭。

## 5. 数据策略

- **默认首运行为空状态**：App 首次启动时不预置任何安全事项、隐患、巡检记录或联系人。列表页展示空状态引导用户创建。
- **本地持久化**：使用 App 本地存储（如 HarmonyOS 的 `preferences` 或关系型数据库）保存用户数据，确保离线可用。
- **可选同步**：未来可扩展云端同步，但当前版本不强制要求网络连接。
- **演示数据说明**：设计阶段 HTML 样稿中可出现示例数据，仅用于视觉参考，不作为默认种子数据写入产品需求。

## 6. 功能需求

### 6.1 安全事项
- 列表展示标题、分类标签、优先级、截止日期、状态。
- 支持按状态（全部/待办/已完成）、按优先级筛选。
- 新增/编辑：标题、分类（消防安全/设备安全/电气安全/其他）、优先级（高/中/低）、负责人、截止日期、备注、状态。
- 删除事项（带确认）。
- 逾期高亮。

### 6.2 风险隐患
- 列表展示隐患标题、位置、风险等级、状态、上报时间。
- 支持按风险等级（高/中/低）和状态筛选。
- 上报隐患：标题、位置、风险等级、描述、照片、上报人。
- 整改进度：状态流转（待处理 → 处理中 → 待验收 → 已完成），每一步可填写备注与时间。
- 删除隐患（带确认）。

### 6.3 巡检打卡
- 巡检任务列表：任务名称、检查点数、完成数、截止时间、状态。
- 进入任务后按检查点逐项确认（正常/异常），异常可拍照与备注。
- 完成全部检查点后提交巡检记录。
- 历史记录列表。

### 6.4 应急联系人
- 联系人卡片：姓名、角色、电话。
- 一键拨打电话、一键发送短信。
- 新增/编辑：姓名、角色、电话、排序。
- 删除联系人（带确认）。

### 6.5 统计看板
- 顶部指标卡片：隐患总数、待整改数、整改完成率、巡检完成率、逾期事项数。
- 趋势图：近 7 天/近 30 天隐患上报数量。
- 分类占比：按隐患等级或事项分类的分布图。
- 时间维度切换：本周/本月/本季度。

## 7. 非目标（Non-Goals）

- 不支持多租户或复杂权限体系（仅区分"管理员/普通员工"开关，默认普通员工）。
- 不接入实时定位后台追踪。
- 不实现消息推送与 IM 即时通讯。
- 不包含 AI 风险识别或自动生成整改方案。
- 不要求云端同步或登录注册流程（本地优先）。

## 8. 验收标准

- [ ] 用户可以从首页进入所有主要功能模块，底部 Tab 切换流畅。
- [ ] 用户可以完成一次完整的安全事项创建、查看、编辑、删除。
- [ ] 用户可以上报隐患并更新整改进度。
- [ ] 用户可以完成一次巡检打卡并查看历史记录。
- [ ] 用户可以新增应急联系人并一键拨号/发短信。
- [ ] 统计看板能反映当前数据状态（至少包含关键指标数字）。
- [ ] 空状态、加载状态、错误状态有清晰提示。
- [ ] App 在目标手机分辨率下视觉层级清晰，底部导航不铺成全宽。

## 9. 开放性/待确认问题

1. 是否需要角色区分（如管理员可指派整改人，普通员工只能上报）？
2. 是否需要预设检查点模板（如每日消防巡检检查点）？
3. 是否需要导出报表（PDF/Excel）？
4. 照片是拍摄新照片还是允许从相册选择？
5. 统计图表是否需要真机图表库，还是先以数字卡片+简单条形图实现？

## 10. Target Device

- **屏幕分辨率**: 1272vp × 2756vp（宽 × 高）
- **设备类型**: 普通手机（非折叠机、非平板）
- **像素密度倍率**: 3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px ≈ 1vp）
- **布局策略**: 内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。**
- **特殊注意**: 2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。
