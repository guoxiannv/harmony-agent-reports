# 今天吃什么 - 产品规格说明书

## 1. 产品概述

### 价值主张
帮助用户快速、有趣地做出"今天吃什么"的日常决策，减少选择焦虑；同时记录每日饮食，生成可分享的打卡卡片，培养健康饮食的仪式感。

### 产品类别
生活决策 / 饮食记录工具类 App。

### 主要用户场景
- 做饭前不知道吃什么，打开 App 用转盘随机决定。
- 查看冰箱里剩下的食材，获得推荐菜谱。
- 根据当前心情（想吃清淡/重口味/快手菜）或时间（早餐/午餐/晚餐/夜宵）筛选菜品。
- 记录当天吃了什么，形成饮食日志。
- 将今日饮食打卡生成卡片，分享给朋友或社交平台。

## 2. 用户核心任务（Jobs-to-be-Done）

1. **随机决策**：通过转盘快速抽取今日菜品，缓解选择困难。
2. **食材推荐**：录入冰箱现有食材，获得可用食材匹配的菜品建议。
3. **情境筛选**：按心情标签与用餐时段筛选菜品池。
4. **饮食打卡**：记录每餐实际吃了什么，形成连续打卡记录。
5. **分享卡片**：将一次打卡或一次转盘结果生成美观卡片并分享。

## 4. 导航模型

- 底部 Tab 导航：首页、食材、发现、打卡、我的。
- 菜品详情从转盘结果、推荐列表、筛选列表、打卡记录均可进入。
- 分享卡片页从打卡记录或转盘结果页跳转进入。
- 返回统一使用系统返回手势 / 顶部返回按钮。

## 5. 数据源策略

- 菜品库：内置本地菜品数据集（名称、标签、所需食材、做法摘要、热量、用餐时段、心情标签）。
- 用户数据：
  - 冰箱食材列表：用户手动增删，本地持久化。
  - 打卡记录：用户每日手动添加，本地持久化。
  - 偏好设置：转盘自定义菜品、主题偏好等，本地持久化。
- 首次启动为真实空状态：
  - 食材列表为空。
  - 打卡记录为空。
  - 转盘默认使用内置菜品池，但允许用户自定义。
- 分享功能：通过 HarmonyOS 系统分享接口分享生成的卡片图片。
- 不依赖后端服务，所有数据本地存储。

## 6. 组件结构

- **BottomTabBar**：底部 Tab 导航。
- **WheelSpinner**：转盘组件，支持旋转动画与结果展示。
- **IngredientInput**：食材输入与标签展示。
- **DishCard**：菜品卡片，展示菜品封面、名称、标签、匹配度/热量。
- **MoodFilter**：心情/时段筛选标签组。
- **CheckinForm**：打卡表单（选择菜品、餐别、备注）。
- **CheckinTimeline**：打卡时间轴列表。
- **ShareCardCanvas**：分享卡片预览与生成。
- **DishDetailPanel**：菜品详情面板（食材、步骤、营养）。
- **EmptyState**：空状态插画与引导。

## 7. 功能需求

### 7.1 随机转盘抽菜
- 首页展示一个可旋转的菜品转盘。
- 转盘菜品池默认为内置菜品。
- 用户点击"开始转盘"后转盘旋转并停在随机菜品。
- 结果展示菜品名称与"查看做法"入口。
- 支持"再转一次"。
- 支持进入转盘设置，添加/删除自定义菜品。

### 7.2 按冰箱食材推荐
- 用户在食材页添加当前冰箱拥有的食材。
- 系统根据食材匹配内置菜品，按匹配度排序展示。
- 支持一键将推荐菜品加入今日打卡。
- 支持从推荐结果进入菜品详情。

### 7.3 按心情/时间筛选
- 发现页提供心情标签（想吃清淡、重口味、快手菜、健康减脂、放纵一下、治愈系）。
- 提供用餐时段标签（早餐、午餐、晚餐、夜宵）。
- 用户可多选标签，实时筛选菜品池。
- 支持重置筛选条件。

### 7.4 每日饮食打卡记录
- 打卡页展示今日已打卡记录，按时间轴排列。
- 用户可以手动添加一次打卡：选择餐别、选择/搜索菜品、填写备注。
- 支持编辑和删除打卡记录。
- 展示今日打卡统计（餐次、总热量）。
- 历史记录以日历/列表形式展示。

### 7.5 分享卡片
- 从打卡记录或转盘结果进入分享卡片页。
- 展示预设模板卡片：日期、菜品名称、一句话心情、打卡天数/连续天数。
- 支持切换卡片模板风格。
- 支持保存卡片到相册和系统分享。

## 8. 非目标

- 不实现社交 Feed、好友关注、评论点赞。
- 不实现食材保质期提醒。
- 不实现在线菜谱搜索或网络爬虫。
- 不实现外卖平台对接。
- 不实现 AI 生成菜品。

## 9. 验收标准

- [ ] 首页转盘可正常旋转并随机返回一个内置菜品。
- [ ] 食材页可添加/删除食材，并按食材推荐菜品。
- [ ] 发现页可按心情/时段筛选菜品。
- [ ] 打卡页可添加/编辑/删除打卡记录，展示今日统计。
- [ ] 分享卡片页可生成卡片并调用系统分享。
- [ ] 底部 Tab 可在各主页面间切换。
- [ ] 应用构建通过，可在模拟器/真机运行。

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即HTML设计稿390px = 设备上1170vp物理像素，但ArkTS使用vp单位时1px≈1vp）
- 布局策略：内容区max-width约束在390-420vp内居中，禁止全宽铺满。**注意：Tab导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与body一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在PC浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header+核心卡片）在顶部600vp内可见
