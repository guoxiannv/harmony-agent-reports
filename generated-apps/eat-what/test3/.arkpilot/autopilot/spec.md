# 今天吃什么 — 产品规格说明书

## 1. 产品概述

### Value Proposition
帮助用户快速、有趣地决定“今天吃什么”。通过随机转盘、冰箱食材智能推荐、心情/时间筛选，降低每日决策疲劳；结合每日饮食打卡与分享卡片，让“吃饭”这件事变得有仪式感和社交趣味。

### Product Category
生活工具 / 饮食决策助手（HarmonyOS 原生应用）。

### Primary User Scenario
- 用户不知道吃什么，打开 App，快速转转盘或输入冰箱里有的食材，得到推荐。
- 用户根据当下心情（想吃好的 / 想减肥 / 想快）或时间（早餐 / 午餐 / 晚餐 / 夜宵）筛选推荐。
- 用户决定后一键打卡，记录饮食，并生成精美分享卡片保存或分享。

## 2. Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 3. Core Jobs-to-be-Done

1. **快速决定吃什么**：通过随机转盘在众多候选菜品中抽取一个结果，提供“再转一次”和“就选这个”的快捷操作。
2. **利用现有食材做推荐**：用户输入或勾选冰箱里有的食材，App 基于本地规则匹配可做的菜品，减少浪费。
3. **按场景筛选**：支持按心情标签（放纵、健康、快手、治愈、解馋）和时间场景（早餐、午餐、晚餐、夜宵、加餐）过滤候选菜品。
4. **记录每日饮食**：对决定好的菜品进行打卡，形成按日/按周的饮食记录，可补录、可删除。
5. **生成分享卡片**：将打卡的菜品自动生成一张视觉卡片，支持保存到本地和系统分享。

## 4. Page Structure

| 页面 | 类型 | 说明 |
|------|------|------|
| 首页 / 决策页（home） | Tab 主页面 | 默认页，包含转盘、推荐入口、筛选器、今日结果 |
| 食材页（pantry） | Tab 主页面 | 管理冰箱食材，按食材推荐菜品 |
| 打卡页（check-in） | Tab 主页面 | 每日饮食记录列表、补录、删除 |
| 我的（profile） | Tab 主页面 | 统计、设置、历史入口 |
| 结果页（result） | 非 Tab 全屏页 | 展示转盘/推荐的结果详情，可跳转打卡 |
| 打卡编辑页（check-in-editor） | 非 Tab 全屏页 | 补录/编辑一次打卡 |
| 分享卡片页（share-card） | 非 Tab 全屏页 | 预览生成的分享卡片，可保存/分享 |

## 5. Navigation Model

- 底部 Tab 导航，固定 4 个 Tab：
  - 决策（tab-home）
  - 食材（tab-pantry）
  - 打卡（tab-check-in）
  - 我的（tab-profile）
- 结果页、打卡编辑页、分享卡片页以全屏覆盖形式从 Tab 页进入；返回后回到进入前的 Tab。
- 分享卡片页可通过打卡页或结果页进入。

## 6. Data Source Strategy

- **本地静态菜品库**：应用内置一份 ArkTS 常量菜品数据集，包含菜品名、所需食材、标签（心情/时间）、推荐语、难度、预估时间。不需要网络接口。
- **用户食材清单**：使用 AppStorage / LocalStorage 持久化用户录入的冰箱食材。
- **打卡记录**：使用 AppStorage / LocalStorage 持久化，每条记录包含日期、餐别、菜品名、心情标签、备注、创建时间。
- **分享卡片**：基于当前打卡数据，在内存中渲染为卡片视图，并调用系统能力保存/分享图片。不需要后端。
- **初始状态**：默认空数据。转盘直接使用内置菜品库；食材清单和打卡记录首次进入为空。

## 7. Component Structure

- **WheelSpinner**：转盘组件，支持起始/停止动画、结果回调。
- **MoodFilter**：心情标签筛选组件，单选/多选标签。
- **TimeFilter**：时间场景筛选组件。
- **PantryInput**：食材输入与标签展示组件。
- **DishCard**：菜品结果卡片，展示菜名、食材、标签、推荐语。
- **CheckInList**：打卡列表组件，按日期分组。
- **CheckInEditor**：打卡编辑表单（日期、餐别、菜品、备注）。
- **ShareCardCanvas**：分享卡片预览组件。
- **BottomTabBar**：底部 Tab 导航（沿用现有组件契约）。

## 8. Functional Requirements

### FR-1 随机转盘
- 首页展示一个可旋转的菜品转盘，至少包含 8 个默认菜品。
- 点击“开始转盘”后转盘加速并减速停止，高亮最终选中的菜品。
- 提供“再转一次”和“就选这个”按钮。

### FR-2 冰箱食材推荐
- 食材页允许用户添加/删除冰箱食材。
- 点击“看看能做什么”后，列出所有可被当前食材匹配的菜品（无需严格匹配，命中主要食材即可）。
- 列表支持按心情/时间进一步筛选。

### FR-3 心情/时间筛选
- 首页和食材页都提供心情标签筛选。
- 首页提供时间场景筛选。
- 筛选结果实时影响转盘候选池或推荐列表。

### FR-4 每日饮食打卡
- 打卡页以日期分组展示历史打卡。
- 支持从结果页一键打卡，也支持手动补录。
- 支持删除单条打卡记录。

### FR-5 分享卡片
- 从打卡详情或结果页可生成分享卡片。
- 卡片包含日期、菜品名、心情标签、一句推荐语、装饰视觉。
- 支持保存图片到相册和系统分享。

## 9. Non-Goals

- 不接入真实 AI 推荐或云端菜谱 API。
- 不做用户账号、登录、云同步。
- 不做复杂的食材图像识别（通过手动输入）。
- 不做社交 Feed 或好友互动。
- 不需要初始预设种子数据；首次打开为空状态。

## 10. Acceptance Criteria

- [ ] 首页转盘可正常旋转并给出结果。
- [ ] 食材页添加/删除食材后，推荐列表正确更新。
- [ ] 心情/时间筛选能影响转盘候选池和推荐列表。
- [ ] 打卡记录可增删查，并按日期分组展示。
- [ ] 分享卡片可预览，并具备保存/分享入口（至少 UI 可点，具体系统能力在 QA 阶段验证）。
- [ ] 底部 Tab 导航在四个主页面间切换正常。
- [ ] 应用在目标手机上构建成功并运行无崩溃。

## 11. Open Questions

- 是否需要在转盘中加入自定义菜品？（当前版本暂不做，后续扩展）
