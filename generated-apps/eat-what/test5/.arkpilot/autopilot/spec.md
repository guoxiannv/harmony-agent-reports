# 今天吃什么 — 产品需求文档

## 摘要

一款面向日常饮食决策的 HarmonyOS 生活助手 App。用户在"不知道吃什么"时，可以通过随机转盘快速得到今日菜品推荐；也可以基于冰箱已有食材获得搭配建议；还能按心情、用餐时段进行筛选。完成用餐后可进行每日饮食打卡，并生成分享卡片记录生活。

## 价值主张

把"今天吃什么"从反复纠结变成一次有趣、有记录、可分享的决策体验。App 不仅提供即时答案，还帮助用户管理冰箱食材、追踪饮食习惯，并通过分享卡片让日常三餐变得更有仪式感。

## 产品类别与主要用户场景

- **产品类别**：生活工具 / 饮食决策助手
- **目标用户**：日常自己做饭或点外卖、希望减少选择困难的年轻用户和家庭用户
- **主要场景**：
  1. 临近饭点没有主意，打开 App 转转盘决定今天吃什么。
  2. 冰箱里还有剩余食材，输入后获得推荐菜品。
  3. 想吃点特别的，按心情或时间筛选后决定。
  4. 吃完后打卡记录，连续打卡养成饮食记录习惯。
  5. 生成精美分享卡片，分享到社交平台。

## 核心任务（Jobs-to-be-Done）

1. **快速决策**：通过随机转盘在一众菜品中抽出一个今日推荐，减少选择焦虑。
2. **食材利用**：录入冰箱现有食材，系统基于食材推荐可做的菜品，减少浪费。
3. **情境筛选**：根据当前心情或用餐时段快速过滤出合适的菜品列表。
4. **饮食打卡**：记录每日三餐内容，形成连续打卡日历和统计。
5. **社交分享**：将今日菜品或打卡记录生成分享卡片，便于保存或分享。

## 页面结构

| 页面 | 页面 ID | 说明 |
|------|---------|------|
| 首页 | `home-page` | 今日状态、快捷入口、最近打卡、转盘入口 |
| 转盘页 | `wheel-page` | 转盘抽菜、选择/编辑候选菜品、查看结果 |
| 食材推荐页 | `ingredient-page` | 管理冰箱食材、按食材推荐菜品 |
| 筛选页 | `filter-page` | 按心情/时段筛选菜品 |
| 打卡页 | `checkin-page` | 每日打卡、日历视图、打卡统计 |
| 分享页 | `share-page` | 生成并预览分享卡片 |
| 菜品管理页 | `dish-page` | 添加、编辑、删除个人菜品库 |

## 导航模型

底部 Tab 导航，包含 4 个一级入口：

1. **首页**（tab-home）
2. **推荐**（tab-recommend）— 聚合转盘、食材、筛选三个入口
3. **打卡**（tab-checkin）
4. **我的**（tab-profile）— 包含菜品管理、设置

推荐页内使用二级分段控制（Segment）切换：转盘 / 食材 / 筛选。

## 数据来源策略

- **本地持久化**：使用 App 本地 Preference / 关系型数据库（RDB）存储：
  - 用户自定义菜品库
  - 冰箱食材清单
  - 每日打卡记录
  - 转盘候选池设置
- **首次启动状态**：不预置默认种子数据。首屏显示空状态引导用户添加第一道菜品或食材。若后续需要演示截图，HTML 设计稿中可填充视觉样本数据，但这不是产品功能要求。
- **推荐算法**：基于本地规则匹配（食材包含度 + 心情/时段标签过滤），无需后端或网络。
- **分享卡片**：基于本地数据，通过 Canvas / 组件截图生成图片文件保存到相册。

## 组件结构

- **全局**
  - `BottomTabBar`：底部导航
  - `TopHeader`：页面标题与返回/操作按钮
  - `LoadingState` / `EmptyState` / `ErrorState`：通用状态组件
- **首页**
  - `TodayStatusCard`：今日已打卡状态与推荐入口
  - `QuickActionGrid`：转盘、食材、筛选快捷入口
  - `RecentCheckinList`：最近打卡摘要
- **转盘**
  - `SpinWheel`：可旋转转盘
  - `WheelResultPanel`：抽中结果展示
  - `CandidateChips`：候选菜品管理
- **推荐/食材**
  - `IngredientInput`：食材输入框
  - `IngredientTagList`：已添加食材标签
  - `DishRecommendationList`：推荐菜品列表
- **筛选**
  - `MoodFilterChips`：心情标签
  - `TimeFilterChips`：用餐时段标签
  - `FilteredDishGrid`：筛选结果网格
- **打卡**
  - `CheckinCalendar`：打卡日历
  - `MealTypeSelector`：早餐/午餐/晚餐
  - `CheckinForm`：打卡内容表单
  - `CheckinStatsPanel`：连续打卡/累计打卡统计
- **分享**
  - `ShareCardPreview`：分享卡片预览
  - `ShareActionBar`：保存/分享操作
- **菜品管理**
  - `DishList`：菜品列表
  - `DishFormSheet`：添加/编辑菜品表单

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）**。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 验收标准

- [ ] 用户能在转盘页添加候选菜品并点击转盘进行抽取，抽中结果明确展示。
- [ ] 用户能在食材页输入冰箱食材，系统基于食材推荐可做菜列表。
- [ ] 用户能按心情/时段筛选，获得匹配的菜品列表。
- [ ] 用户能在打卡页选择日期、餐别并记录内容，日历上正确显示打卡标记。
- [ ] 用户能在分享页生成包含今日菜品或打卡信息的卡片，并触发保存/分享。
- [ ] 用户能在菜品管理页增删改查个人菜品库。
- [ ] 所有数据在 App 生命周期内持久化，重启后仍然存在。
- [ ] 构建通过，App 能在目标设备上正常启动并运行。

## 非目标（Out of Scope）

- 不接入第三方菜谱 API 或网络搜索。
- 不提供社区、关注、点赞等社交功能。
- 不提供营养成分计算或热量追踪。
- 不预置默认菜品/食材种子数据。

## 开放问题

1. 是否需要为转盘添加音效或震动反馈？（建议作为后续增强）
2. 分享卡片是否包含二维码/小程序跳转？（建议首版不包含）
3. 菜品库是否支持从相册导入图片？（建议首版使用默认占位图或纯色卡片）
