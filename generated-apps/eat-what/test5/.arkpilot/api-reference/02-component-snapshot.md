## Module Notes

- **Deprecation**: `componentSnapshot.get()` and `componentSnapshot.createFromBuilder()` (top-level module functions) are deprecated since API 18. The recommended approach (since API 12) is to obtain a `ComponentSnapshot` instance via `uiContext.getComponentSnapshot()` and call instance methods.
- **API version floor**: Module supported from API 10 (Stage model only). UIContext-based APIs start at API 12.
- **Import**: `import { componentSnapshot } from '@kit.ArkUI'` for the module; `import { UIContext } from '@kit.ArkUI'` for UIContext-based snapshots; `import { image } from '@kit.ImageKit'` for `PixelMap`; `import { colorSpaceManager } from '@kit.ArkGraphics2D'` for color space options.
- **Limitations**: Snapshot only captures the component's own area (overlapping siblings not captured). Transforms on the target component itself are not reflected. XComponent/Video/camera streaming scenarios should use `createPixelMapFromSurface` instead. Offscreen snapshots have ~500ms delay. `getSync` blocks the main thread with a 3s timeout.
- **Error codes**: 401 (parameter error), 100001 (invalid id/builder), 160001 (image not ready), 160002 (timeout), 160003 (unsupported color space/dynamic range), 160004 (isAuto(true) not supported for offscreen snapshots).
- **SystemCapability**: `SystemCapability.ArkUI.ArkUI.Full`.
- **SnapshotSizeLimitation** is available from API version 26.0.0.
