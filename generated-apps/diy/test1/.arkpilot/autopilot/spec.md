# 手作·印记 — Product Specification

## 1. Elevator Pitch

为独立手作工作室与工坊打造的会员管理应用。通过温润的陶土色系界面与手写感设计，让店主在记录会员档案、追踪积分、预约课程、提醒沉睡会员、扣减材料库存时，都像在翻阅一本手账本一样自然。

## 2. Product Category & Primary User Scenario

- **产品类别**：线下手作工坊会员管理工具（会员档案 + 积分 + 课程预约 + 库存联动）。
- **主要用户**：手作工作室主理人、课程老师、工坊前台。
- **核心场景**：
  1. 新会员到店，快速建立档案并记录擅长工艺与作品照片。
  2. 会员消费或签到后自动累计积分，积分可用于兑换课程或材料。
  3. 会员预约手作课程，主理人在日历视图中管理日程。
  4. 系统根据规则识别沉睡会员，在主界面给出温馨提醒。
  5. 课程消耗材料时，自动从库存中扣减并记录。

## 3. Core Jobs-to-be-Done

1. **管理会员档案**：新增/编辑会员信息、上传作品图、标注擅长工艺（陶艺/编织/木工/皮具/染织等）。
2. **积分运营**：消费/签到获得积分、积分兑换、查看积分流水与余额。
3. **课程预约与日历**：会员预约课程、店主确认/取消预约、按日/周查看课程安排。
4. **沉睡会员唤醒**：设置沉睡阈值（如 60 天未到店），首页显示待唤醒会员列表与一键提醒入口。
5. **库存联动扣减**：维护材料库存、课程与材料清单关联、预约/上课后自动扣减库存并生成记录。

## 5. Navigation Model

- **主导航**：底部 Tab Bar，共 5 个一级入口：
  1. 首页（tab-home）
  2. 会员（tab-members）
  3. 课程（tab-calendar）
  4. 积分（tab-points）
  5. 库存（tab-inventory）
- **次级导航**：
  - 首页顶部的"今日"快捷入口进入会员列表、课程日历或沉睡会员提醒。
  - 会员列表 → 会员详情（点击卡片）。
  - 会员详情 → 编辑会员 / 积分流水 / 预约历史。
  - 课程日历 → 课程详情（点击日程卡片）。
  - 课程详情 → 编辑课程 / 报名学员 / 材料消耗。
  - 库存管理 → 出入库记录 / 库存扣减记录。
- **模态/弹窗**：
  - 新增会员、签到、积分兑换、预约确认、库存扣减确认可使用底部 Sheet 或全屏页面。

## 6. Data Source Strategy

- **数据持久化**：使用 HarmonyOS 应用级 Preferences 或关系型数据库（RDB）本地存储。
- **数据模型**：
  - `Member`：会员 ID、姓名、手机号、头像、擅长工艺、作品图数组、积分余额、最后到店时间、创建时间。
  - `PointRecord`：记录 ID、会员 ID、类型（earn / redeem）、来源（消费/签到/兑换课程/兑换材料）、积分变化值、关联订单/课程 ID、时间。
  - `Course`：课程 ID、标题、老师、时间、时长、容量、已报名人数、关联材料清单、状态。
  - `Booking`：预约 ID、课程 ID、会员 ID、状态（pending / confirmed / cancelled / completed）、预约时间。
  - `InventoryItem`：材料 ID、名称、分类、单位、当前库存、安全库存、备注。
  - `InventoryLog`：记录 ID、材料 ID、类型（in / out / auto-deduct）、数量、关联课程 ID、关联会员 ID、时间、备注。
  - `Settings`：沉睡阈值天数、积分规则 JSON、提醒时间。
- **空态策略**：
  - 应用首次启动时所有列表为空，显示空状态插画与引导按钮。
  - HTML 设计稿中可出现示例数据，但仅用于视觉参考，不作为功能种子数据。

## 7. Component Structure

- **Foundation**
  - 陶土色主题 Token、手写感字体栈、圆角/阴影体系。
- **Layout**
  - 安全区容器、底部 Tab 导航、顶部标题栏、滚动内容区。
- **Data Display**
  - 会员卡片、作品图网格、积分流水项、课程卡片、日程时间轴、库存项、统计数字卡片。
- **Input**
  - 文本输入、手机号输入、标签选择器（擅长工艺）、图片选择器、日期时间选择器、数字步进器。
- **Feedback**
  - Toast、确认对话框、空状态、加载骨架、Badge 角标。
- **Navigation**
  - TabBar、返回按钮、底部 Sheet、页面跳转。

## 8. Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。**
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见

## 9. Acceptance Criteria

1. 底部 Tab 导航可在 5 个主页面间切换。
2. 会员列表支持查看、搜索、新增、编辑、删除会员。
3. 会员详情展示档案、擅长工艺标签、作品图网格、积分余额与流水。
4. 签到/消费可为会员增加积分；积分可兑换课程或材料并扣减余额。
5. 课程日历支持按日/周查看，点击课程进入详情，可管理报名学员。
6. 系统根据设定的未到店天数自动标记沉睡会员，首页提供提醒入口。
7. 课程消耗材料时，库存自动扣减并生成扣减记录。
8. 所有列表在首次启动时为空，显示空状态。
9. 视觉风格符合陶土色 + 手写感，界面温润手作。

## 10. Non-Goals

- 不实现云端同步或多设备登录。
- 不实现支付收款功能（仅记录消费金额与积分）。
- 不实现复杂的员工权限系统（单店主视角）。
- 不实现图片拍摄与滤镜编辑（仅支持从相册选择）。
- 不实现社交分享或评价功能。

## 11. Open Questions

1. 是否需要支持会员头像与作品图的批量删除？（建议：单张删除即可，保持轻量。）
2. 积分规则是否允许店主自定义消费金额与积分的兑换比例？（建议：默认 1 元 = 1 积分，可配置。）
3. 沉睡会员提醒是否需要推送通知，还是仅在应用内 Badge 提醒？（建议：先应用内提醒。）
