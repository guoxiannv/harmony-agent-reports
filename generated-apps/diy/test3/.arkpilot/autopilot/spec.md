# 手作·印记 — 产品规格说明书

## 价值主张

为手工坊、陶艺/木作/皮具体验空间打造一款温润的会员管理 App。
让店主和会员都能「摸得到温度」：
- 会员：一份记录着自己擅长工艺与作品图的电子手账，能预约课程、签到攒积分、查看消费与余额。
- 店主：一个轻量 CRM，自动识别沉睡会员并提醒，课程预约与库存自动联动，减少漏单与手工记账。

## 产品分类与核心场景

- **产品类别**：生活服务 / 小微企业会员管理工具（面向手工坊的会员端）。
- **主要用户场景**：
  1. 到店签到，累积积分。
  2. 浏览课程日历，预约/取消体验课。
  3. 维护个人档案：上传作品、标注擅长工艺。
  4. 店主后台查看沉睡会员提醒，发送唤醒权益。
  5. 课程耗材库存扣减与预警。

## 核心 Jobs-to-be-Done

1. **记住我**：会员档案集中保存个人标签、擅长工艺与作品集，随时翻看成长痕迹。
2. **省得记**：消费记录、积分余额、签到天数自动汇总，不用翻聊天/账本。
3. **约得上**：课程日历直观展示可约时段，一键预约并同步扣减名额/耗材库存。
4. **不被忘**：系统按规则识别沉睡会员，自动提醒店主并生成唤醒任务。
5. **管得住**：库存与课程绑定，开课后自动扣减对应耗材，库存不足时拦截预约。

## 导航模型

- **底部 Tab 导航**：首页、课程、积分、我的（共 4 个主 Tab）。
- **二级页面**：课程详情、会员编辑、沉睡会员提醒、库存管理通过路由跳转进入，不占用 Tab。
- 返回统一使用页面左上角返回按钮或系统返回手势。

## 数据源策略

- 使用 App 本地 `AppStorage` / `PersistentStorage` 作为第一数据源，保证离线可用。
- 会员、积分、课程预约、库存、沉睡会员提醒均以本地 JSON/关系化模型存储。
- 初始状态为「真实空态」：不预置演示数据，首次进入显示空列表与引导文案。
- 图片数据：作品图使用本地 `fileUri` 或沙箱临时路径；不强制要求后端。
- 后续可扩展：通过 `http` 模块对接门店后台，但本期不实现网络同步。

## 组件结构

- **原子组件**：`ClayButton`、`ClayInput`、`ClayCard`、`ClayTag`、`ClayAvatar`、`ClayIcon`。
- **业务组件**：
  - `CheckInButton`：签到按钮与动画反馈。
  - `PointCard`：积分余额与今日可获积分。
  - `CraftTagGrid`：擅长工艺标签编辑/展示。
  - `WorkGallery`：作品图网格与上传入口。
  - `ClassCalendar`：周/月日历与可约标识。
  - `ClassSlotCard`：单节课卡片（时间/名额/耗材）。
  - `DormantMemberRow`：沉睡会员行，带唤醒按钮。
  - `InventoryItemRow`：库存项行，带预警色标。
- **页面**：每个路由对应一个 `Page` 包裹的 ArkTS 页面。

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390–420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。**
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。

## 非目标

- 不实现支付、退款、收银台。
- 不实现多门店/多角色权限体系。
- 不实现社交分享与评论。
- 不接入第三方登录或推送服务。

## 验收标准

1. 会员可完成签到，积分余额与连续签到天数正确更新。
2. 会员可编辑档案、添加/删除擅长工艺标签、上传作品图。
3. 课程日历能显示可约与已约状态，预约成功后名额与库存同步扣减。
4. 库存不足时，对应课程预约按钮置灰并显示库存不足提示。
5. 沉睡会员列表按「超过 30 天未签到/未消费」自动计算，店主可标记已唤醒。
6. 构建通过（`build_project` 无错误），首屏在 600vp 内可见核心内容。

## 待澄清问题

- 课程取消规则：课前多久可取消、取消后库存是否回滚？（当前假设课前 24 小时可取消，名额回滚，耗材预扣回滚。）
