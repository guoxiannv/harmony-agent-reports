# 手作·印记 App — 产品规格说明书

## 1. Value Proposition

“手作·印记”是一款面向手作工坊（陶艺、木作、皮具、花艺等）的会员经营助手。它以会员档案与作品沉淀为核心，连接积分激励、课程预约、沉睡唤醒和材料库存扣减，帮助小工作室把散落的客户信息、预约记录、作品图片和物料消耗整合为一条温润、可感知的客户关系主线，让每位客人都能被“记住”，让每次创作都有迹可循。

## 2. Product Category & Primary User Scenario

- **产品类别**：本地生活服务 / 小型工作室会员管理 + 课程预约工具。
- **主要用户**：手作工坊主理人、课程助教、前台接待。
- **核心场景**：
  1. 客人到店后，快速查看其历史作品、擅长工艺、积分余额和可用权益，推荐匹配课程。
  2. 预约课程后，系统自动扣减对应材料库存，并在库存不足时提醒补货。
  3. 对长期未到店的沉睡会员，一键发送专属召回优惠或作品展示，唤回再次到店。
  4. 每日开店时，主理人签到打卡并累积积分，形成轻量运营闭环。

## 3. Core Jobs-to-be-Done

1. **会员建档与作品沉淀**：为每位会员建立档案，记录昵称、联系方式、擅长工艺、兴趣标签、作品集（图片），支持新增/编辑/搜索/筛选。
2. **积分体系运营**：通过到店签到、消费、完成课程等行为获得积分；积分可用于兑换课程优惠或材料包；系统展示积分明细与余额。
3. **课程预约与日历管理**：主理人/会员可查看课程表、预约时段、取消预约；课程与会员档案、积分、库存关联。
4. **沉睡会员自动提醒**：基于“距上次到店/消费超过设定阈值（默认 45 天）”自动筛选沉睡会员，展示列表并支持发送唤醒消息/优惠券。
5. **库存关联扣减**：课程预约或消费发生时，按课程/商品配置自动扣减对应材料库存；库存不足时阻止操作并提示补货。

## 4. Page Structure

| Page ID | 页面名称 | 说明 |
|---------|----------|------|
| `home` | 首页 | 今日概览：待上课、待处理预约、积分动态、沉睡提醒入口、快捷签到/消费 |
| `members` | 会员列表 | 会员搜索、筛选、新增入口、会员卡片列表 |
| `member-detail` | 会员详情 | 会员信息、擅长工艺、作品集、积分明细、消费/签到记录、沉睡状态标签 |
| `points` | 积分中心 | 当前积分余额、积分规则说明、积分明细、兑换入口 |
| `courses` | 课程日历 | 日历视图 + 课程列表；可切换日期、查看预约状态 |
| `course-detail` | 课程详情 | 课程信息、已预约会员、剩余名额、关联材料、预约/取消按钮 |
| `inventory` | 库存管理 | 材料列表、当前库存、库存预警、补货入口、扣减记录 |
| `sleeping-members` | 沉睡会员 | 自动筛选出的沉睡会员列表、唤醒操作、批量提醒 |
| `check-in` | 签到/消费 | 选择会员、签到或录入消费金额、积分增减、关联课程/材料扣减 |
| `profile` | 我的 | 当前登录主理人信息、积分规则设置、沉睡阈值设置、通知开关 |

## 5. Navigation Model

- **底部 Tab 导航**（5 个主入口）：
  1. 首页 (`tab-home`)
  2. 会员 (`tab-members`)
  3. 课程 (`tab-courses`)
  4. 库存 (`tab-inventory`)
  5. 我的 (`tab-profile`)
- **二级页面**：会员详情、课程详情、沉睡会员、积分中心、签到/消费，通过页面路由 Push 进入，统一左上角返回。
- **快捷操作**：首页提供“签到/消费”FAB/快捷按钮，一键进入签到/消费页面。

## 6. Data Source Strategy

- **优先真实空状态**：首次启动时，数据库/持久化存储为空，不内置任何示例会员、课程或库存。所有列表初始显示空状态插图与新建引导。
- **本地持久化**：使用 HarmonyOS 应用级/preferences 持久化或本地关系型数据库存储会员、课程、预约、积分流水、库存、配置项。
- **离线优先**：所有读写操作先操作本地数据；网络同步为可选非必须。
- **示例数据仅限视觉**：设计阶段 HTML 中可放置视觉示例，但不得把这些数据作为产品功能上的默认种子数据。

## 7. Functional Behavior

### 7.1 会员档案

- 必填字段：昵称、手机号。
- 可选字段：生日、擅长工艺（多选标签：陶艺/木作/皮具/花艺/银饰/布艺等）、兴趣标签、备注。
- 作品集：支持从相册选择 1~9 张作品图，本地存储路径/缩略图，详情页可查看大图。
- 搜索：支持按昵称/手机号模糊搜索。
- 筛选：按擅长工艺标签筛选。

### 7.2 积分体系

- 积分获取：
  - 每日签到：+5 分（每人每天限一次）。
  - 消费：每消费 1 元 +1 分（可配置比例）。
  - 完成课程：按课程积分奖励。
- 积分使用：
  - 兑换课程抵扣：在预约时选择使用积分。
  - 兑换材料包：扣减库存并生成积分流水。
- 积分流水：记录时间、类型（签到/消费/课程奖励/兑换）、数值、关联会员、备注。
- 规则可配置：签到积分、消费比例、沉睡阈值，在“我的-设置”中修改。

### 7.3 课程预约与日历

- 课程模型：名称、时间、容量、已预约人数、关联材料及消耗量、积分奖励、价格。
- 日历视图：按日展示课程；选择日期后下方列出当天课程。
- 预约：选择会员后预约，校验容量与库存；成功后扣减库存并加积分奖励（若配置）。
- 取消：可取消预约，释放容量，并回滚库存与积分（如已发放）。
- 预约状态：未开始 / 进行中 / 已完成 / 已取消。

### 7.4 沉睡会员自动提醒

- 判定规则：距会员最后一次到店（签到/消费/课程完成）超过阈值（默认 45 天）即标记为沉睡。
- 列表展示：沉睡天数、上次到店时间、历史作品、擅长工艺。
- 唤醒动作：
  - 发送“想念你”专属消息（应用内/短信分享二选一）。
  - 发放一张积分优惠券或体验课券。
  - 一键拨打/复制联系方式。
- 支持批量选择多个沉睡会员进行群发唤醒。

### 7.5 库存关联扣减

- 材料模型：名称、规格、单位、当前库存、安全库存阈值、所属工艺。
- 扣减时机：课程预约成功、积分兑换材料包、手动录入消耗。
- 预警：当前库存 ≤ 安全库存时，页面显示预警标签；库存不足时阻止预约并提示补货。
- 补货：手动修改库存数量，并记录补货流水。

## 8. Acceptance Criteria

1. 首页能展示今日待处理事项、快捷签到、沉睡会员提醒入口，空状态友好。
2. 会员列表支持搜索、筛选、新增，详情页展示完整档案与作品集。
3. 积分中心正确显示余额、规则、明细，签到/消费/课程奖励后实时更新。
4. 课程日历能按日查看课程，完成预约/取消，库存随预约联动扣减/回滚。
5. 沉睡会员页面自动筛选并展示沉睡会员，支持唤醒操作。
6. 库存不足时，阻止消耗类操作并给出补货引导。
7. 所有首屏关键内容（Header + 核心卡片）位于顶部 600vp 内可见。
8. 底部 Tab 导航固定在底部，不超出内容最大宽度，禁止侧边导航。

## 9. Non-Goals

- 不接入第三方支付或线上收银。
- 不实现多门店/多账号协作。
- 不实现复杂的报表与数据分析。
- 不实现实时消息推送（仅应用内提醒与系统分享）。
- 不要求首次启动内置示例数据。

## 10. Open Questions

- 是否需要会员自主预约（C 端小程序）？本版本仅支持主理人端操作。
- 是否需要照片云备份？本版本仅本地存储。
- 是否需要多语言？本版本仅简体中文。

## 11. Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。
