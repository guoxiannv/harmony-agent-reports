# 手作·印记 App 产品规格

## 产品一句话

为手工坊、陶艺教室、手作工作室打造的会员管理与课程预约助手，让每位会员的“手作印记”都被温柔记录。

## 产品分类与核心场景

- **产品分类**：生活服务 / 垂直行业工具（手工坊会员管理）。
- **主要用户**：手工坊店主、课程老师、工作室运营者。
- **核心场景**：
  1. 到店后快速查看会员档案与作品记录。
  2. 课前查看课程预约与日历排期。
  3. 每日签到/消费后积分变动与沉睡提醒。
  4. 课程耗材或作品库存的关联扣减与预警。

## 核心 Jobs-to-be-Done

1. **管理会员档案** — 记录会员昵称、联系方式、擅长工艺、入会时间、标签与作品图片。
2. **积分体系运营** — 通过消费、签到等行为累积积分，并展示积分余额与流水。
3. **课程预约与日历** — 浏览可预约课程，完成预约/取消，按日历查看排期。
4. **沉睡会员唤醒** — 自动识别超过阈值未互动的会员，生成提醒列表。
5. **库存联动扣减** — 课程/作品制作时自动扣减关联耗材库存，低库存给出提示。

## 页面结构

| 页面 | 功能定位 | 导航位置 |
|------|----------|----------|
| 首页（Home） | 今日概览：待上课提醒、签到入口、沉睡会员预警、积分快讯 | 底部 Tab 首页 |
| 会员页（Members） | 会员列表、搜索/筛选、新建会员入口 | 底部 Tab 会员 |
| 会员详情页（MemberDetail） | 档案、擅长工艺、作品图、积分流水、编辑 | 从会员页进入 |
| 课程页（Courses） | 课程列表、课程预约/取消、日历视图切换 | 底部 Tab 课程 |
| 课程详情/预约页（CourseBooking） | 课程详情、时间选择、库存耗材提示、确认预约 | 从课程页进入 |
| 积分页（Points） | 积分余额、积分规则、获取/消耗流水 | 底部 Tab 积分 |
| 库存页（Inventory） | 耗材库存列表、入库/出库、低库存预警 | 底部 Tab 库存 |
| 我的页（Profile） | 工作室信息、提醒设置、数据导出入口 | 底部 Tab 我的 |

## 导航模型

- 底部 5 个一级 Tab：首页、会员、课程、积分、库存、我的。
  - 实际规划为 5 个：首页、会员、课程、积分、我的；库存管理入口置于首页快捷卡片 + 我的页，作为高频但非一级 Tab 的模块，避免底部过多。
  - **修正**：考虑库存是核心需求，底部 Tab 为 5 项：首页、会员、课程、库存、我的。积分体系渗透到首页与会员详情中，不单独作为一级 Tab，减少导航复杂度。
- 次级页面通过 NavPathStack 或 router 跳转：MemberDetail、CourseBooking、PointDetail（可选）。
- 模态：签到成功、积分变动、新建会员、库存出入库。

## 数据源策略

- 本地优先（PersistentStorage / AppStorage + 本地 JSON/关系型存储），无需云端账号即可使用。
- 数据模型：
  - `Member`：id, name, phone, avatar, craftTags[], joinDate, works[], points, note。
  - `Work`：id, title, craft, imageUris[], createDate。
  - `PointsRecord`：id, memberId, type(earn|redeem), amount, reason, timestamp。
  - `Course`：id, title, dateTime, durationMin, capacity, bookedMemberIds[], materialId, materialQtyPerPerson。
  - `CourseBooking`：id, memberId, courseId, status, createdAt。
  - `InventoryItem`：id, name, unit, qty, warningThreshold, courseLinks[]。
- 初始状态为空，不预设示例数据；允许用户手动添加。
- 沉睡判定：基于最后消费/签到/预约时间，阈值可配置（默认 30 天）。
- 库存扣减：预约确认时校验耗材，取消预约时回滚。

## 组件结构

- `BottomTabBar`：底部导航，含徽章点（沉睡提醒、低库存）。
- `MemberCard`：会员头像、昵称、积分、标签、最近互动时间。
- `WorkGallery`：作品图网格/横向滚动，支持点击查看。
- `PointsBadge`：积分余额胶囊。
- `PointsHistoryItem`：积分流水项。
- `CourseCard`：课程封面、时间、余量、预约按钮。
- `CalendarStrip` / `CalendarMonth`：日历视图切换。
- `InventoryItemCard`：库存名称、当前量、阈值进度条、预警色。
- `SleepReminderBanner`：沉睡会员预警横幅。
- `SignInButton`：今日签到按钮（含已签到状态）。

## 非目标

- 不涉及线上支付、收银台、退款逻辑。
- 不涉及多店铺/多账号协同。
- 不涉及云端同步与备份（可后续扩展）。
- 不内置示例会员/课程/库存数据。

## 验收标准

1. 能创建、编辑、删除会员，并上传/关联作品图片。
2. 能为会员签到、记录消费并正确变动积分。
3. 能浏览课程、预约课程、在日历中查看排期。
4. 沉睡会员超过阈值时首页出现提醒，可跳转查看列表。
5. 课程预约确认时自动扣减关联库存，取消时回滚。
6. 低库存物品在库存页与首页给出视觉预警。
7. 构建通过，能在模拟器/真机运行。

## 开放问题

1. 作品图片是否支持拍照/从相册选择？建议支持 `photoAccessHelper` / picker。
2. 积分规则是否允许自定义（签到+5、消费每元+1 等）？建议在我的页提供规则设置。
3. 库存是否支持扫码入库？当前版本不纳入，后续扩展。

## Target Device

- 屏幕分辨率：1272vp × 2756vp（宽 × 高）
- 设备类型：普通手机（非折叠机、非平板）
- 像素密度倍率：3x（即 HTML 设计稿 390px = 设备上 1170vp 物理像素，但 ArkTS 使用 vp 单位时 1px≈1vp）
- 布局策略：内容区 max-width 约束在 390-420vp 内居中，禁止全宽铺满。**注意：Tab 导航栏等必须在底部，且若使用 `position: fixed` 必须限制宽度与 body 一致（如 `width: 100%; max-width: 390px; margin: 0 auto; left: 0; right: 0;`），严禁在 PC 浏览器上全宽拉伸。严禁生成在侧边（如右边/左边）。**
- 特殊注意：2756px 高度意味着屏幕很长，滚动区域充足，但需确保首屏关键内容（Header + 核心卡片）在顶部 600vp 内可见。
