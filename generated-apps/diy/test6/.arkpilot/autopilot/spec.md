# 手作·印记 App — Product Spec

## 1. Product Summary

**App Name:** 手作·印记（Handmade Mark）  
**Category：** 手作工作室会员管理与课程预约工具  
**Platform：** HarmonyOS（普通手机）  
**Primary User Scenario：** 陶艺、木工、皮具、布艺等手作工作室的经营者，用于管理会员档案、记录消费与积分、安排课程预约、追踪物料库存，并在会员长期未到店时自动提醒激活。

**Value Proposition：**  
将散落的会员信息、消费记录、课程预约、积分余额和物料库存统一在一款温润、手写感的手作风格 App 中，让经营者能随手查看每位会员的“手作印记”，减少沉睡会员流失，并通过积分与课程体系提升复购与粘性。

## 2. Core Jobs-to-be-Done

1. **会员建档与画像管理** — 记录会员姓名、联系方式、擅长工艺、过敏/注意事项与代表作品图片，形成可快速检索的会员画像。
2. **消费与签到积分体系** — 记录每笔消费金额、积分增减、签到打卡，支持手动补录与余额展示。
3. **课程预约与日历管理** — 创建课程（名称、时间、容量、消耗物料），会员可预约/取消，日历视图展示每日课程与剩余名额。
4. **沉睡会员自动提醒** — 设定未到店阈值（默认 30 天），系统自动标记沉睡会员并推送提醒，支持一键发送唤醒通知。
5. **库存关联扣减** — 课程或消费与物料 SKU 绑定，发生预约/消费时按预设配比自动扣减库存，低库存时提醒补货。

## 5. Data Source Strategy

- **本地优先（LocalPreference + RelationalStore）**：会员、课程、预约、积分、库存数据首先存储在设备本地，保证离线可用与首屏加载速度。
- **空首跑状态**：首次安装不预置任何会员、课程或库存数据；所有示例数据仅在 HTML 设计稿中以视觉占位形式出现，不作为产品功能需求。
- **云同步（可选，非 MVP）**：后续阶段可考虑通过 AppGallery Connect 云函数/云数据库实现多设备同步，但不在当前交付范围内。
- **图片存储**：会员作品图使用应用私有文件目录，通过相机/相册选择后保存本地缩略图路径。

## 6. Functional Requirements

### 6.1 会员档案
- 支持新增、编辑、删除会员。
- 字段：姓名（必填）、手机号（必填，去重）、昵称、生日、擅长工艺（多选/自定义）、备注、过敏/注意事项、作品图（最多 6 张）。
- 列表支持按姓名/手机号搜索，支持沉睡会员筛选。
- 详情页展示消费总额、积分余额、最近到店、已预约课程、作品图墙。

### 6.2 消费/签到积分体系
- 积分规则：消费 1 元 = 1 积分（可配置），每日签到 +5 积分。
- 支持手动补录：消费补录（金额、说明）与积分直接增减（奖励/扣减）。
- 积分流水展示时间、类型、数值、余额。
- 签到按钮每日只能点击一次，成功后有动画反馈。

### 6.3 课程预约与日历管理
- 课程字段：名称、日期时间、时长、容量、价格、地点、关联物料及消耗量、已预约人数。
- 日历模式：月视图，日期圆点标记有课的日期；列表模式：按时间倒序。
- 预约：选择会员，校验容量与库存，成功后扣减库存并增加消费/积分记录。
- 取消预约：释放名额，库存回滚。

### 6.4 沉睡会员自动提醒
- 默认阈值 30 天未到店即标记为“沉睡”。
- 首页展示沉睡会员数量卡片；会员列表支持沉睡筛选。
- 点击唤醒可发送本地通知（或记录唤醒动作），并支持备注唤醒方式。

### 6.5 库存关联扣减
- 物料字段：名称、规格、单位、当前库存、安全库存、备注。
- 课程创建/编辑时可绑定消耗物料与数量；预约成功自动扣减。
- 低库存（≤安全库存）时在首页与库存页显示预警。
- 支持手动入库、出库、盘点调整。

## 7. Non-Goals

- 多门店/多员工权限管理
- 线上支付与收银台
- 会员端用户自主预约（本版本仅服务经营者）
- 云端数据同步与备份
- 营销短信/微信推送（使用本地通知）

## 8. Acceptance Criteria

1. 可在目标分辨率手机上完成新增会员、上传作品图、搜索会员的完整闭环。
2. 签到/消费后积分余额实时更新，流水可追溯。
3. 创建课程并预约会员后，关联物料库存正确扣减，低库存触发首页提醒。
4. 会员超过阈值未到店后自动进入沉睡列表，首页展示提醒并可一键唤醒。
5. 课程日历视图可切换月份，点击日期进入当日课程/预约。
6. 所有页面底部 Tab 导航完整，底部固定不拉伸。
